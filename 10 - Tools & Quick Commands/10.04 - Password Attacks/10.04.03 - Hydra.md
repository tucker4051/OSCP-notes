
## Overview

THC Hydra is a fast network login cracker used for:

- brute force attacks
- dictionary attacks
- credential spraying
- password auditing

Supports many protocols including:

```text
SSH, FTP, RDP, SMB, HTTP, HTTPS, Telnet,
VNC, MySQL, PostgreSQL, SMTP, POP3, LDAP
```

---

## Basic Syntax

```bash
hydra [options] <service>://<target>
```

---

## Common Flags

| Flag | Description |
|---|---|
| `-l` | Single username |
| `-L` | Username list |
| `-p` | Single password |
| `-P` | Password list |
| `-s` | Custom port |
| `-t` | Number of threads |
| `-V` | Verbose output |
| `-vV` | Very verbose |
| `-f` | Stop after first success |
| `-o` | Save output to file |
| `-R` | Restore interrupted session |
| `-I` | Ignore restore file |
| `-W` | Wait time between retries |
| `-u` | Loop users instead of passwords |

# SSH

## Single User

```bash
hydra -l george -P rockyou.txt ssh://192.168.50.201
```

## Custom Port

```bash
hydra -l george -P rockyou.txt -s 2222 ssh://192.168.50.201
```

## Multiple Users

```bash
hydra -L users.txt -P rockyou.txt ssh://192.168.50.201
```

# HTTP Basic Auth

```bash
hydra -l admin -P rockyou.txt http-get://192.168.50.50/protected
```

## <Base64(username:password)>

The Base64 string is the encoding of `username:password`

When using Hydra for HTTP Basic Auth brute force: - Use the `http-get` or `http-head` module. 
- Provide the username with `-l`.
- Provide the password list with `-P`. 
- Hydra automatically encodes credentials in Base64 and sends the Authorization header.

Example command: ```bash hydra -l admin -P passwords.txt target.com http-get /

## Faster

Be careful with:

- account lockouts
- IPS/IDS

# OPSEC / Detection Notes

Hydra is noisy.

Blue teams may detect:

- repeated failed logins
- brute force attempts
- authentication spikes
- abnormal login behaviour

Common protections:

- Fail2Ban
- account lockouts
- rate limiting
- MFA
- geo-blocking
