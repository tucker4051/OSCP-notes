## Overview

Windows systems support multiple remote management protocols that allow administrators to manage hosts remotely.

These protocols are frequently encountered during internal assessments and often allow:

- remote command execution
- credential validation
- lateral movement
- privilege escalation
- full system access

Common Windows remote management protocols:

| Protocol | Purpose |
|----------|---------|
| RDP | GUI remote desktop access |
| WinRM | remote command execution |
| WMI | remote management & execution |

Remote management is enabled by default on many modern Windows Server systems.

---

# Remote Desktop Protocol (RDP)

## Overview

RDP allows graphical remote access to Windows systems.

Useful for:

- interactive login
- GUI-based enumeration
- manual credential hunting
- executing tools directly on host

---

## Common Ports

| Port | Protocol |
|------|----------|
| 3389 | TCP |
| 3389 | UDP |

---

## Nmap Enumeration

Basic scan:

```bash
nmap -sV -sC -p3389 <target>
```

RDP scripts:

```bash
nmap -p3389 --script rdp* <target>
```

Packet tracing:

```bash
nmap -p3389 --script rdp* --packet-trace <target>
```

---

## RDP Security Check Script

Tool from Cisco CX Security Labs:

```bash
git clone https://github.com/CiscoCXSecurity/rdp-sec-check.git

cd rdp-sec-check

./rdp-sec-check.pl <target>
```

Can identify:

- NLA requirements
- TLS settings
- encryption level
- security configuration

---

## Connecting to RDP (Linux)

### xfreerdp

```bash
xfreerdp /u:<user> /p:"<password>" /v:<target>
```

Pass the hash:

```bash
xfreerdp /u:<user> /pth:<hash> /v:<target>
```

Ignore certificate:

```bash
xfreerdp /u:<user> /p:"<password>" /v:<target> /cert:ignore
```

## Alternative RDP clients

```text
rdesktop
remmina
xfreerdp
```

## Detection Note

Nmap uses:

```text
mstshash=nmap
```

This cookie may be detected by:

- EDR
- IDS
- SIEM tools

Can trigger:

- alerts
- account lockouts
- blocking rules

# Windows Remote Management (WinRM)

## Overview

WinRM is a command-line based remote management protocol.

Uses:

- SOAP over HTTP/HTTPS
- PowerShell Remoting
- remote command execution

Frequently used for lateral movement.

---

## Common Ports

| Port | Protocol |
|------|----------|
| 5985 | HTTP |
| 5986 | HTTPS |

Older versions used:

```text
80
443
```

---

## Nmap Enumeration

```bash
nmap -sV -sC -p5985,5986 <target>
```

Avoid DNS resolution:

```bash
nmap -sV -sC -p5985,5986 --disable-arp-ping -n <target>
```

---

## Testing WinRM Connectivity (PowerShell)

```powershell
Test-WsMan <target>
```

---

## Connecting with evil-winrm

```bash
evil-winrm -i <target> -u <user> -p <password>
```

Pass-the-hash:

```bash
evil-winrm -i <target> -u <user> -H <hash>
```

Useful for:

- PowerShell access
- file upload/download
- credential harvesting
- privilege escalation
- AD enumeration

---

## WinRS (Windows Remote Shell)

Allows remote command execution via WinRM.

Example:

```cmd
winrs -r:<target> -u:<user> -p:<password> "whoami"
```

# Windows Management Instrumentation (WMI)

## Overview

WMI provides deep administrative control of Windows systems.

Used for:

- remote command execution
- system information gathering
- process control
- service management
- lateral movement

WMI is widely used by administrators and attackers.

---

## Ports Used

| Port | Description |
|------|-------------|
| 135 | RPC endpoint mapper |
| dynamic ports | assigned after connection |

After connecting on 135, communication shifts to a high random port.

---

## Remote Command Execution (Impacket)

Example:

```bash
wmiexec.py <user>:"<password>"@<target> "<command>"
```

Example:

```bash
wmiexec.py admin:"Password123"@10.10.10.5 hostname
```

---

## Alternative Impacket WMI tools

```text
wmiexec.py
wmipersist.py
dcomexec.py
atexec.py
```

1. scan for open ports

```bash

5985
5986
135
```

2. identify protocol

3. test credentials

4. attempt remote access

5. enumerate host

6. search for credentials

7. escalate privileges

# Tips

RDP often reveals:

- saved credentials
- configuration files
- KeePass databases
- scripts
- browser sessions

WinRM is very common in:

- domain environments
- admin workstations
- server environments

WMI is frequently allowed internally even when WinRM is restricted.
