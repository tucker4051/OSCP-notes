## Purpose

Authentication and access control attacks target weaknesses in how an application identifies users, manages sessions, authorises actions, and restricts access to data or functionality.

This section contains notes for access control weaknesses such as IDOR, HTTP verb tampering, and PHP type juggling.

---

## Core Idea

Authentication answers:

> Who are you?

Access control answers:

> What are you allowed to do?

Vulnerabilities occur when an application trusts user input, object identifiers, request methods, roles, tokens, or comparison logic incorrectly.

## Common Authentication Issues

Authentication issues may include:

- weak login logic
- default credentials
- predictable passwords
- weak password reset flows
- session fixation
- insecure remember-me tokens
- insecure token comparison
- type juggling issues
- missing MFA enforcement
- username enumeration
- insecure logout behaviour

## Common Access Control Issues

Access control issues may include:

- IDOR
- BOLA
- missing object ownership checks
- role bypass
- forced browsing
- horizontal privilege escalation
- vertical privilege escalation
- HTTP verb tampering
- parameter tampering
- mass assignment
- hidden function access
- unauthorised API object access

---

## Testing Flow

1. Identify authentication and session behaviour.
2. Identify user roles.
3. Map accessible functionality.
4. Test object identifiers.
5. Test horizontal access between users.
6. Test vertical access between roles.
7. Test alternate HTTP methods.
8. Test predictable IDs and mass enumeration.
9. Test token and comparison logic where relevant.
10. Document the access control failure and impact.

---

## High-Value Targets

Focus on functionality involving:

- user profiles
- account settings
- invoices
- orders
- documents
- messages
- tickets
- admin functions
- password reset
- user management
- API object access
- exports and downloads
- approval workflows

---

## Common Impact

Authentication and access control flaws can lead to:

- unauthorised data access
- user impersonation
- account takeover
- privilege escalation
- admin function access
- sensitive document exposure
- API data exposure
- modification of another user’s data
