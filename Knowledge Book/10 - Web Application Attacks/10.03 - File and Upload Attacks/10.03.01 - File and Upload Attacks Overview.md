## Purpose

File and upload attacks involve abusing how an application reads, includes, uploads, stores, writes, or executes files.

This section links together file inclusion, file upload, and web shell-related material.

Detailed exploitation notes live primarily under `03.02 - Manual Exploitation` and `04.02 - Web Shells`.

---

## Core Idea

File handling becomes dangerous when user input can influence:

- what file is read
- what file is included
- what file is uploaded
- where a file is stored
- what extension is accepted
- whether a file is executed
- whether a file can be overwritten
- whether a file can be accessed through the web server

The key question is:

> Can file handling be converted into disclosure, code execution, or access?

## Common File Attack Types

|Attack Type|Description|
|---|---|
|Local File Inclusion|Read or include local files from the target|
|Remote File Inclusion|Include attacker-hosted remote files|
|File Upload Abuse|Upload files that bypass validation|
|Web Shell Upload|Upload server-side code for command execution|
|Path Traversal|Access files outside the intended directory|
|Arbitrary File Read|Read sensitive files from the server|
|Arbitrary File Write|Write files to sensitive or executable locations|
|Upload plus LFI|Include uploaded payload through file inclusion|

---

## Common Targets

Useful files and locations include:

- web configuration files
- application configuration files
- CMS configuration files
- database connection files
- `.env` files
- source code
- backup files
- logs
- SSH keys
- uploaded file directories
- web root directories
- temporary directories

---

## File and Upload Attack Flow

1. Identify file-related functionality.
2. Check whether user input controls file paths or names.
3. Test for file read or inclusion.
4. Test upload restrictions.
5. Identify storage and execution behaviour.
6. Try bypasses where appropriate.
7. Look for sensitive file disclosure.
8. Attempt web shell or RCE path where valid.
9. Document the exact path and impact.

---

## Common Impact

File and upload attacks can lead to:

- sensitive file disclosure
- source code disclosure
- credential disclosure
- web shell upload
- reverse shell trigger
- command execution
- admin panel compromise
- privilege escalation clues
- initial access
