## Purpose

Windows pivoting covers techniques for routing traffic through Windows hosts or using Windows-native tools to forward traffic.

Use this when the pivot host is Windows, SSH is unavailable, or Windows-specific tooling is required.

Windows pivoting may involve:

- Netsh port forwarding
- Plink SSH tunnelling
- Chisel on Windows
- SocksOverRDP
- RDP-based pivoting
- Windows firewall considerations
- Windows-native routing behaviour

---

## Core Idea

A compromised Windows host may have access to internal networks that the attacker cannot reach directly.

Because Windows targets often lack standard Linux tools, pivoting may require Windows-specific techniques or portable binaries.

The key question is:

> What Windows-compatible method can route traffic through this host?

## Common Windows Pivoting Methods

|Method|Use Case|
|---|---|
|Netsh portproxy|Native Windows port forwarding|
|Plink|SSH tunnelling from Windows|
|Chisel|Reverse/forward SOCKS tunnels|
|SocksOverRDP|SOCKS proxy through RDP|
|Meterpreter|Framework route and port forwarding|
|RDP|Interactive access and internal tooling|

---

## When to Use Windows Pivoting

Use Windows pivoting when:

- [ ]  the pivot host is Windows
- [ ]  RDP access is available
- [ ]  WinRM access is available
- [ ]  SSH is not available on the pivot
- [ ]  internal services are reachable only from Windows host
- [ ]  Windows-native port forwarding is needed
- [ ]  firewall rules affect direction of traffic
- [ ]  portable tools can be transferred and executed

---

## Windows Pivoting Workflow

1. Gain access to Windows host.
2. Identify network interfaces and routes.
3. Identify internal networks and reachable services.
4. Choose a Windows-compatible pivoting method.
5. Transfer required tooling if needed.
6. Configure the tunnel or port forward.
7. Test connectivity.
8. Route tools through the pivot.
9. Document the setup.

## Windows-Specific Checks

Check:

- [ ]  Windows version
- [ ]  Current user privileges
- [ ]  Local firewall rules
- [ ]  Listening ports
- [ ]  Network interfaces
- [ ]  Routing table
- [ ]  DNS settings
- [ ]  Proxy settings
- [ ]  Tool execution restrictions
- [ ]  Defender/AV behaviour
- [ ]  File transfer method

---

## Common Mistakes

- [ ]  Forgetting Windows firewall rules
- [ ]  Forgetting admin rights may be needed for some portproxy actions
- [ ]  Using Linux-only tooling assumptions
- [ ]  Not checking whether the pivot can reach the target
- [ ]  Not checking whether the attacker can reach the pivot
- [ ]  Forgetting to transfer the correct architecture binary
- [ ]  Not documenting portproxy rules
- [ ]  Leaving stale forwarding rules behind
