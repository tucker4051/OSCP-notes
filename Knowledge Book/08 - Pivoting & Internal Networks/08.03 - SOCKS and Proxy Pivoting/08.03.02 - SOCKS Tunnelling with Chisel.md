## Overview

**Chisel** is a fast TCP/UDP tunnelling tool written in **Go** that transports traffic over **HTTP**, secured via **SSH-like encryption**.

Chisel is extremely useful in restricted environments where:

- direct network access is blocked
- outbound traffic is limited to HTTP/HTTPS
- SSH is unavailable
- we need a SOCKS proxy pivot
- firewall rules prevent direct connectivity

Chisel allows creation of:

- SOCKS5 proxies
- reverse tunnels
- port forwards
- HTTP-based covert channels

---

# Why Chisel is Powerful

Many corporate environments allow outbound traffic on:

```
TCP 80 (HTTP)
TCP 443 (HTTPS)
```

Chisel leverages this by tunnelling traffic over HTTP while encrypting communication.

This allows pivoting even when:

- VPN access is unavailable
- inbound firewall rules block connections
- network segmentation prevents lateral movement

# Scenario

Internal network:

```
172.16.5.0/23
```

Domain Controller:

```
172.16.5.19
```

Pivot host:

```
Ubuntu server
10.129.202.64
172.16.5.x
```

Goal:

Access the Domain Controller from the attack host through the pivot host using a SOCKS5 tunnel.

---

# Architecture

Chisel operates in a client-server model:

| Component | Runs on | Purpose |
|----------|--------|---------|
| chisel server | pivot or attack host | listens for connections |
| chisel client | attack or pivot host | connects to server |
| SOCKS5 proxy | local attack host | forwards traffic through tunnel |

# Install Chisel

Clone repository:

```bash
git clone https://github.com/jpillora/chisel.git
```

Build binary:

```bash
cd chisel
go build
```

Output binary:

```
./chisel
```

---

# Binary Compatibility Note

Differences in **glibc versions** between systems may cause execution issues.

Solutions:

- compile on similar OS version
- download prebuilt binary from GitHub Releases
- use statically compiled binary

---

# Transfer Binary to Pivot Host

```bash
scp chisel ubuntu@10.129.202.64:~/
```

Example:

```text
chisel 100% 11MB 1.2MB/s 00:09
```

---

# Start Chisel Server on Pivot Host

```bash
./chisel server -v -p 1234 --socks5
```

---

# Parameter Breakdown

| Parameter | Description |
|----------|-------------|
| server | run in server mode |
| -v | verbose output |
| -p 1234 | listening port |
| --socks5 | enable SOCKS5 proxy |

# Configure Proxychains

Edit configuration:

```bash
sudo nano /etc/proxychains.conf
```

Add:

```bash
socks5 127.0.0.1 1080
```

Verify:

```bash
tail -f /etc/proxychains.conf
```

Example:

```text
[ProxyList]

socks5 127.0.0.1 1080
```

---

# Pivot to Internal Domain Controller

Use proxychains:

```bash
proxychains xfreerdp /v:172.16.5.19 /u:victor /p:pass@123
```

Traffic flow:

```
xfreerdp
     ↓
proxychains
     ↓
SOCKS5 proxy (127.0.0.1:1080)
     ↓
Chisel client
     ↓
HTTP tunnel
     ↓
Chisel server (pivot host)
     ↓
172.16.5.19
```

# Reverse Chisel Pivot

In some scenarios:

- inbound connections to pivot host are blocked
- pivot host can only make outbound connections

Solution:

use reverse tunnelling.

---

# Start Chisel Server on Attack Host

```bash
sudo ./chisel server --reverse -v -p 1234 --socks5
```

Output:

```text
Reverse tunnelling enabled
Listening on http://0.0.0.0:1234
```

```
127.0.0.1:1080
```

# Chisel Traffic Flow

## Normal Mode

```
attack host --> pivot host
client        server
```

## Reverse Mode

```
pivot host --> attack host
client       server
```

# Advantages of Chisel

## Works over HTTP

Bypasses many firewall restrictions.

## Encrypted transport

Secure communication channel.

## Supports TCP and UDP

More flexible than some tunnelling tools.

## SOCKS5 proxy support

Compatible with proxychains.

## Reverse tunnelling capability

Useful when inbound firewall rules block pivot host.

## Lightweight binary

Single executable.

---

# Limitations

## Requires binary transfer

May trigger EDR alerts.

## Requires proxychains

Manual configuration needed.

## HTTP traffic may be inspected

Deep packet inspection can detect anomalies.

## Potential version compatibility issues

glibc mismatch can cause runtime errors.

# Mental Model

Chisel creates an encrypted HTTP tunnel that carries SOCKS traffic.

Instead of:

```
attacker --> internal network
```

traffic flows through:

```
attacker
   ↓
SOCKS proxy
   ↓
HTTP tunnel
   ↓
pivot host
   ↓
internal network
```
