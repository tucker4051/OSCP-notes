## Purpose

Meterpreter pivoting uses a Meterpreter session to route traffic, forward ports, or access internal networks through a compromised host.

Use this when exploitation has produced a Meterpreter session and you need to reach internal systems through it.

Meterpreter pivoting can support:

- route addition
- internal subnet access
- port forwarding
- SOCKS proxying
- post-exploitation routing
- framework-assisted lateral movement

---

## Core Idea

Meterpreter can act as a pivot point inside Metasploit.

Once a route is added through a session, Metasploit modules can target internal hosts through that session.

The key question is:

> Can my Meterpreter session route traffic to the internal network?

## When to Use Meterpreter Pivoting

Use Meterpreter pivoting when:

- [ ]  a Meterpreter session exists
- [ ]  the compromised host can reach internal networks
- [ ]  Metasploit modules will be used against internal targets
- [ ]  a quick route through the session is needed
- [ ]  port forwarding through the session is useful
- [ ]  SOCKS proxy support is needed through Metasploit

---

## Common Meterpreter Pivoting Actions

|Action|Purpose|
|---|---|
|Add route|Route Metasploit module traffic through session|
|Port forward|Forward a specific internal port|
|SOCKS proxy|Route external tools through Metasploit SOCKS|
|Background session|Keep session active while using modules|
|Internal scanning|Run Metasploit modules against internal hosts|

---

## Meterpreter Pivoting Workflow

1. Gain Meterpreter session.
2. Enumerate interfaces and routes on target.
3. Identify internal subnets.
4. Add route through the session.
5. Confirm route is active.
6. Test internal connectivity.
7. Use Metasploit modules or SOCKS proxy.
8. Document route and session dependency.

---

## Common Mistakes

- [ ]  Forgetting the session must stay alive
- [ ]  Adding the wrong route/subnet
- [ ]  Not checking target routes before pivoting
- [ ]  Confusing Meterpreter routes with system routes
- [ ]  Running external tools without SOCKS/proxy setup
- [ ]  Scanning too aggressively through a Meterpreter route
- [ ]  Not documenting which session provides the route
- [ ]  Losing the session and breaking the pivot
