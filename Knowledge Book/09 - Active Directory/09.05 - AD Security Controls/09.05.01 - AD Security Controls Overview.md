## Purpose

AD security control enumeration identifies defensive technologies, policies, and configuration settings that may affect enumeration, credential attacks, exploitation, lateral movement, and privilege escalation in a Windows domain environment.

Use this section before running noisy tools, spraying passwords, attempting relay attacks, or relying on specific AD attack paths.

---

## Core Idea

Security controls influence which techniques are likely to work, fail, or create risk.

Before choosing an AD attack path, identify controls such as:

- endpoint protection
- Defender
- EDR
- PowerShell logging
- AMSI
- AppLocker
- Constrained Language Mode
- SMB signing
- LDAP signing
- LAPS
- password policy
- account lockout policy
- firewall rules
- local admin restrictions
- NTLM restrictions
- audit policies

The goal is to avoid guessing and choose techniques that match the environment.

## Why Security Controls Matter

Security controls may affect:

|Activity|Relevant Controls|
|---|---|
|Password spraying|Password policy, lockout policy, monitoring|
|LLMNR/NBT-NS poisoning|LLMNR/NBT-NS settings, endpoint protection|
|NTLM relay|SMB signing, LDAP signing, EPA/channel binding|
|PowerShell tooling|AMSI, Constrained Language Mode, Script Block Logging|
|Tool execution|Defender, EDR, AppLocker, WDAC|
|Lateral movement|Firewall, local admin restrictions, WinRM/RDP policy|
|Credential dumping|LSASS protection, Defender, EDR, privilege level|
|Enumeration|Logging, network segmentation, LDAP restrictions|

## Common Controls to Enumerate

### Endpoint and Execution Controls

Check for:

- Windows Defender
- third-party antivirus
- EDR
- AppLocker
- Windows Defender Application Control
- PowerShell Constrained Language Mode
- AMSI
- script block logging
- module logging
- transcription logging

---

### Authentication and Credential Controls

Check for:

- password policy
- account lockout threshold
- lockout duration
- password complexity
- password history
- NTLM restrictions
- cached credential policy
- Credential Guard
- LSASS protection
- local administrator restrictions
- LAPS

---

### Network and Protocol Controls

Check for:

- SMB signing
- LDAP signing
- LDAPS usage
- EPA/channel binding
- firewall rules
- WinRM availability
- RDP availability
- DNS behaviour
- LLMNR status
- NBT-NS status
- network segmentation
- proxy settings

---

### Logging and Monitoring

Check for:

- PowerShell logging
- command-line process auditing
- Sysmon
- Windows Event Forwarding
- EDR telemetry
- failed logon monitoring
- account lockout monitoring
- script execution logging
- suspicious service creation alerts

---

## Security Control Enumeration Flow

1. Identify current host and domain context.
2. Check endpoint protection.
3. Check PowerShell restrictions and logging.
4. Check application control.
5. Retrieve password and lockout policy.
6. Check NTLM, SMB, and LDAP-related settings.
7. Check local firewall and remote access services.
8. Check LAPS or local admin management.
9. Identify controls that affect planned techniques.
10. Adjust attack path accordingly.

## Technique Impact Guide

|Control Found|Impact|
|---|---|
|Strong lockout policy|Password spraying must be slow or avoided|
|SMB signing required|SMB relay attacks are restricted|
|LDAP signing required|LDAP relay paths are restricted|
|Defender active|Payloads/tools may be blocked|
|AppLocker/WDAC active|Execution may be restricted|
|Constrained Language Mode|PowerShell capability reduced|
|AMSI active|Script content may be scanned|
|LAPS deployed|Local admin password reuse reduced|
|Credential Guard enabled|Credential dumping may be harder|
|LLMNR disabled|Poisoning opportunity reduced|

---

## Prioritisation

When security controls are present:

1. Prefer built-in enumeration where possible.
2. Avoid noisy tooling early.
3. Use valid credentials carefully.
4. Respect lockout policy.
5. Choose techniques that match allowed protocols.
6. Reassess relay, spraying, and payload-based attacks.
7. Document controls as findings or constraints.

## Common Mistakes

- [ ]  Spraying without checking lockout policy
- [ ]  Attempting relay without checking SMB/LDAP signing
- [ ]  Running tools before checking execution controls
- [ ]  Ignoring PowerShell logging
- [ ]  Ignoring Defender or EDR
- [ ]  Assuming all Windows hosts have the same controls
- [ ]  Ignoring LAPS
- [ ]  Ignoring local firewall rules
- [ ]  Not documenting controls that explain failed techniques
- [ ]  Treating failed tooling as a technical error instead of a control issue
