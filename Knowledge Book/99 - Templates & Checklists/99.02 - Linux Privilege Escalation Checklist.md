# Linux Privilege Escalation Checklist

## Purpose

A quick checklist of areas to investigate after gaining access to a Linux host.

Use this to track what has been checked, what looks interesting, and what needs follow-up.

---

## 1. Initial Context

- [ ] Current user identified
- [ ] User ID and group memberships checked
- [ ] Hostname identified
- [ ] Linux distribution and version checked
- [ ] Kernel version checked
- [ ] System architecture checked
- [ ] Current shell type checked
- [ ] Restricted shell ruled out or escaped
- [ ] Environment variables reviewed
- [ ] PATH reviewed
- [ ] Working directory reviewed

---

## 2. User and Group Review

- [ ] All local users reviewed
- [ ] Users with valid login shells identified
- [ ] Current user home directory reviewed
- [ ] Other accessible home directories reviewed
- [ ] Root home directory access checked
- [ ] Current user group memberships reviewed
- [ ] Interesting groups checked

Interesting groups to look for:

- [ ] `sudo`
- [ ] `adm`
- [ ] `docker`
- [ ] `lxd` / `lxc`
- [ ] `disk`
- [ ] `shadow`
- [ ] `backup`
- [ ] `staff`
- [ ] `systemd-journal`
- [ ] Web service groups such as `www-data`

---

## 3. Sudo Rights

- [ ] Sudo permissions checked
- [ ] `NOPASSWD` entries checked
- [ ] Commands runnable as root checked
- [ ] Commands runnable as another user checked
- [ ] Wildcards in sudo rules reviewed
- [ ] Sudo environment behaviour reviewed
- [ ] Sudo version reviewed
- [ ] Sudo commands checked against GTFOBins
- [ ] Ability to control command arguments checked
- [ ] Ability to control files used by sudo commands checked

---

## 4. Credential Hunting

- [ ] Shell history files checked
- [ ] SSH keys checked
- [ ] SSH `authorized_keys` checked
- [ ] SSH `known_hosts` checked
- [ ] Web application config files checked
- [ ] Database config files checked
- [ ] Environment files checked
- [ ] Backup files checked
- [ ] Old config files checked
- [ ] Hidden files and directories checked
- [ ] Passwords searched in user directories
- [ ] Passwords searched in web directories
- [ ] Passwords searched in `/opt`
- [ ] Passwords searched in `/etc`
- [ ] API keys searched for
- [ ] Tokens searched for
- [ ] Private keys searched for
- [ ] Reused credentials tested where appropriate

---

## 5. Web Application Files

- [ ] Web root directories identified
- [ ] Application directories reviewed
- [ ] Upload directories checked
- [ ] Writable web directories checked
- [ ] Application config files reviewed
- [ ] Database credentials checked
- [ ] Framework secrets checked
- [ ] CMS config files checked
- [ ] Source code reviewed for credentials
- [ ] Source code reviewed for command execution paths
- [ ] Source code reviewed for file write paths
- [ ] Source code reviewed for scheduled/background jobs

## 6. Network and Local Services

- [ ] Network interfaces checked

- [ ] Localhost-only services checked
- [ ] Internal-only services checked
- [ ] Active network connections reviewed
- [ ] ARP/neighbour table checked
- [ ] Unusual ports investigated
- [ ] Root-owned local services identified
- [ ] Databases listening locally checked
- [ ] Admin panels listening locally checked
- [ ] Docker socket checked
- [ ] Internal web apps checked

---

## 7. Running Processes

- [ ] Running processes reviewed
- [ ] Root-owned processes reviewed
- [ ] Process command lines checked for credentials
- [ ] Custom scripts or binaries identified
- [ ] Processes running from unusual locations checked
- [ ] Processes running from writable locations checked
- [ ] Background jobs identified
- [ ] Monitoring or backup agents checked
- [ ] Process environment variables considered
- [ ] Repeating processes monitored

## 8. Cron Jobs and Scheduled Tasks

- [ ] Writable service binaries checked
- [ ] Writable config files checked
- [ ] Writable backup locations checked
- [ ] Writable web directories checked
- [ ] Writable PATH directories checked
- [ ] Writable library paths checked

## 10. SUID and SGID

- [ ] SUID binaries identified
- [ ] SGID binaries identified
