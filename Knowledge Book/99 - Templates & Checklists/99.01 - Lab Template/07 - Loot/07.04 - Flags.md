# Windows Privilege Escalation Checklist

## Purpose

A quick checklist of areas to investigate after gaining access to a Windows host.

Use this to track what has been checked, what looks interesting, and what needs follow-up.

---

## 1. Initial Context

- [ ] Current user identified
- [ ] Hostname identified
- [ ] Domain/workgroup membership checked
- [ ] Operating system version checked
- [ ] Architecture checked
- [ ] Patch level considered
- [ ] Current shell type checked
- [ ] Interactive/non-interactive session confirmed
- [ ] Integrity level checked
- [ ] Current privileges reviewed
- [ ] Current groups reviewed
- [ ] Environment variables reviewed
- [ ] PATH reviewed
- [ ] Working directory reviewed

---

- [ ] Current user reviewed
- [ ] Local users reviewed
- [ ] Local administrators identified
- [ ] Logged-on users checked
- [ ] Recently logged-on users checked
- [ ] Domain users identified where applicable
- [ ] Local groups reviewed
- [ ] Current user group memberships reviewed
- [ ] Interesting local groups checked
- [ ] Disabled or stale accounts noted
- [ ] Home/user profile directories reviewed

- [ ] `Administrators`
- [ ] `Remote Desktop Users`
