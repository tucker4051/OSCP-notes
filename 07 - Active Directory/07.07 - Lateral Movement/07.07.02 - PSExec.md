## Overview

**PsExec** is a Sysinternals tool that can execute processes on remote Windows systems.

It is intended for legitimate remote administration, but it can also be used for lateral movement when valid administrative credentials are available.

The basic idea:

```text
Valid admin credentials
+
ADMIN$ share access
+
File and Printer Sharing
=
Remote process execution
```

In this lesson, PsExec is used to obtain an interactive command shell on a remote host.

---

# What PsExec Does

When PsExec runs against a remote host, it typically performs these actions:

```text
1. Connects to the remote ADMIN$ share.
2. Copies psexesvc.exe to the remote Windows directory.
3. Creates a temporary service on the remote host.
4. Starts that service.
5. Runs the requested command as a child process of psexesvc.exe.
6. Provides an interactive console back to the operator.
```

Plain English:

```text
PsExec copies a helper service to the target, starts it, and uses that service to run your command remotely.
```

---

# Required Conditions

PsExec needs three main conditions.

## 1. Local Administrator Rights

The user authenticating to the target must be a member of the target’s local Administrators group.

Example:

```text
corp\jen is local administrator on FILES04
```

## 2. ADMIN$ Share Available

PsExec needs access to:

```text
\\TARGET\ADMIN$
```

This maps to the remote Windows directory, usually:

```text
C:\Windows
```

## 3. File and Printer Sharing Enabled

File and Printer Sharing must be enabled so the remote admin share can be accessed.

On modern Windows Server systems, this is commonly enabled by default.

---

# Ports and Protocols

PsExec commonly relies on SMB.

Important ports:

```text
TCP 445
TCP 139
```

Most modern Windows environments use:

```text
TCP 445
```

Why SMB matters:

```text
PsExec uses SMB to copy the service binary to the ADMIN$ share.
```

# Basic PsExec Syntax

```powershell
.\PsExec64.exe -i \\<target> -u <domain\user> -p <password> <command>
```

Meaning:

| Option | Meaning |
|---|---|
| `-i` | Run interactively |
| `\\<target>` | Remote host |
| `-u` | Username |
| `-p` | Password |
| `<command>` | Command to execute remotely |

# Why This Works

`corp\jen` has administrative access to `FILES04`.

Because of that, PsExec can:

```text
write to ADMIN$
create a service
start the service
run cmd.exe through that service
```

The resulting remote command shell runs under the supplied account context.

In this example:

```text
corp\jen
```

```text

  |

  v
FILES04 ADMIN$ share
  |
  | psexesvc.exe copied to C:\Windows
  v
Service Control Manager
  |
  | temporary service created and started
  v
psexesvc.exe
  |
  | launches cmd.exe
  v
Interactive shell returned to operator
```

# PsExec vs WMI vs WinRM

| Technique | Main Requirement | How It Executes |
|---|---|---|
| PsExec | Local admin + ADMIN$ + SMB | Copies service and starts it |
| WMI | Local admin + WMI/RPC | Creates process with WMI |
| WinRM | Admin or Remote Management Users + WinRM | Runs commands through WinRM |

Plain English:

```text
PsExec uses SMB and services.
WMI uses RPC and Win32_Process.
WinRM uses remote shell / PowerShell remoting.
```

# When To Use PsExec

PsExec is useful when:

```text
you have local admin credentials
SMB is reachable
ADMIN$ is accessible
WinRM is unavailable
you want an interactive shell without a reverse callback
```

In this lesson, PsExec avoids needing:

```text
a Kali listener
a reverse shell payload
PowerShell payload encoding
```

# Quick Validation Checklist

Before running PsExec, check:

```text
[ ] Do I have valid credentials?
[ ] Is the user local admin on the target?
[ ] Is SMB reachable on TCP 445?
[ ] Is ADMIN$ accessible?
[ ] Is File and Printer Sharing enabled?
[ ] Is PsExec available on the source host?
```

After running PsExec, verify:

```text
[ ] Did the command shell open?
[ ] Does hostname show the target?
[ ] Does whoami show the expected user?
```

# Troubleshooting

## Access Is Denied

Possible causes:

```text
user is not local admin on target
wrong password
UAC/token issue
ADMIN$ unavailable
SMB blocked
service creation blocked
EDR/AV blocking PsExec
```

Check:

```text
credential validity
local admin rights
SMB reachability
ADMIN$ access
```

---

## Cannot Connect to ADMIN$

Possible causes:

```text
SMB blocked
File and Printer Sharing disabled
ADMIN$ share disabled
firewall blocking TCP 445
target offline
name resolution issue
```

Test:

```cmd
dir \\FILES04\ADMIN$
```

---

## PsExec Starts but No Shell Appears

Possible causes:

```text
interactive flag issue
service failed to start
AV/EDR blocked psexesvc.exe
command exited immediately
network session interrupted
```

Try a simple command first:

```powershell
.\PsExec64.exe \\FILES04 -u corp\jen -p Nexus123! cmd /c hostname
```

---

## Hostname Is Not FILES04

If `hostname` does not show the target:

```text
you are not in the remote shell
command ran locally
connection failed or dropped
```

Always verify:

```cmd
hostname
whoami
```

# Detection and Defensive Notes

## PsExec Detection Opportunities

PsExec commonly leaves recognizable activity:

```text
SMB connection to ADMIN$
file write of psexesvc.exe
remote service creation
service start event
psexesvc.exe execution
cmd.exe or other command as child of psexesvc.exe
administrator logon to remote host
```

Common artifacts:

```text
psexesvc.exe
temporary PsExec service
remote service creation logs
SMB admin share access
```

Suspicious process relationship:

```text
psexesvc.exe -> cmd.exe
```

or:

```text
psexesvc.exe -> powershell.exe
```

---

## Defensive Controls

Reduce PsExec abuse by:

```text
limiting local administrator rights
disabling unnecessary ADMIN$ access where appropriate
restricting SMB between workstations and servers
monitoring service creation events
monitoring ADMIN$ writes
blocking known PsExec binaries where appropriate
using tiered administration
avoiding reuse of admin credentials
enabling endpoint detection for remote service creation
```

---

# Key Takeaways

- PsExec is a Sysinternals remote execution tool.
- It can be used for lateral movement when valid local admin credentials are available.
- PsExec requires local admin rights on the target.
- PsExec also needs the ADMIN$ share and File and Printer Sharing.
- PsExec copies `psexesvc.exe` to the remote Windows directory.
- It creates and starts a remote service.
- The requested command runs as a child of `psexesvc.exe`.
- `-i` starts an interactive session.
- Always verify remote execution with `hostname` and `whoami`.
- PsExec is often easier than reverse shells because it can provide a direct interactive shell.
- PsExec is also noisy and commonly detected through SMB, service creation, and `psexesvc.exe` artifacts.
