## Purpose

Active Directory is Microsoft’s directory service for managing users, computers, groups, permissions, authentication, and access to resources in a Windows domain environment.

This section covers Active Directory enumeration, password policy discovery, password spraying, poisoning attacks, security control enumeration, credential attacks, Kerberos attacks, privilege escalation, lateral movement, and future AD topics.

Use this section when working against domain-joined Windows environments or internal networks where Active Directory is present.

---

## Core Idea

Active Directory is built around identity and trust.

A successful AD attack path usually involves understanding:

- domain structure
- users
- groups
- computers
- authentication mechanisms
- password policies
- security controls
- privileges
- trust relationships
- credential exposure
- lateral movement paths
- domain escalation opportunities

The goal is to move from basic domain awareness to a clear attack path.

## Typical Active Directory Attack Flow

1. Confirm domain context.
2. Identify domain name and domain controllers.
3. Enumerate users, groups, computers, and shares.
4. Retrieve password policy.
5. Identify security controls.
6. Build a valid target list.
7. Attempt controlled password spraying where appropriate.
8. Capture authentication material where poisoning is viable.
9. Crack or reuse captured credentials.
10. Enumerate with valid credentials.
11. Identify privilege escalation or lateral movement paths.
12. Move toward domain-level compromise where in scope.

## Password Policy and Spraying

Password spraying is often a key early AD technique, but it must be controlled.

Before spraying, identify:

- password policy
- lockout threshold
- lockout duration
- observation window
- valid usernames
- likely password patterns
- spray timing
- account lockout risk

## Poisoning Attacks

LLMNR and NBT-NS poisoning can be used to capture Net-NTLMv2 authentication attempts when systems attempt to resolve unknown names.

This can support:

- hash capture
- offline cracking
- credential reuse
- relay opportunities, depending on controls
