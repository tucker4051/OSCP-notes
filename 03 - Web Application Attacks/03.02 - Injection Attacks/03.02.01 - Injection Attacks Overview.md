## Purpose

Injection attacks occur when user-controlled input is interpreted as part of a command, query, document, or instruction by a backend system.

This section acts as the web attack index for injection-style vulnerabilities.

Some detailed exploitation notes live elsewhere in the vault and are linked from here.

---

## Core Idea

Injection vulnerabilities happen when data is treated as executable logic.

The affected backend may be:

- operating system shell
- SQL database
- XML parser
- LDAP query
- NoSQL query
- template engine
- command interpreter
- application expression language

The attacker’s goal is to change the intended behaviour of that backend system.

## Common Injection Targets

Look for injection opportunities in:

- URL parameters
- POST body fields
- JSON parameters
- XML input
- cookies
- headers
- search fields
- login forms
- file names
- report generators
- admin panels
- API endpoints
- import/export functions
- backend service integrations

---

## Common Signs of Injection

Possible signs include:

- unexpected errors
- syntax errors
- database errors
- XML parser errors
- command output
- time delays
- response differences
- HTTP 500 responses
- authentication bypass
- unexpected data returned
- blind behavioural differences

---

## Injection Testing Flow

1. Identify user-controlled input.
2. Determine where the input is processed.
3. Identify the backend context if possible.
4. Test for safe behavioural changes.
5. Check for errors, delays, output, or side effects.
6. Confirm the injection type.
7. Determine impact.
8. Move to the relevant detailed note.
9. Convert confirmed impact into access where possible.

---

## Injection Impact

Injection attacks may lead to:

|Impact|Examples|
|---|---|
|Command execution|Command injection, SQL-to-RCE|
|Data extraction|SQL injection, XXE file disclosure|
|Authentication bypass|SQL injection, type confusion|
|File read|XXE, SQL file read, LFI-like paths|
|File write|SQL file write, command injection|
|Credential disclosure|Database dumps, config reads|
|Initial access|Reverse shell, web shell, admin login|
