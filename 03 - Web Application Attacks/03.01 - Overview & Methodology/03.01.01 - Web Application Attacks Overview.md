## Suggested Web Testing Flow

1. Identify the application and technology stack.
2. Check exposed panels, CMSs, plugins, versions, and files.
3. Review default credentials and weak configuration.
4. Test injection attacks.
5. Test file and upload functionality.
6. Test client-side issues.
7. Test authentication and access control.
8. Test API-specific behaviour if APIs are present.
9. Test server-side parser or backend behaviour.
10. Convert confirmed vulnerabilities into access where possible.

---

## High-Value Findings

Prioritise findings that may lead to:

- command execution
- file read
- file write
- authentication bypass
- admin panel access
- credential disclosure
- sensitive data exposure
- web shell upload
- session theft
- privilege escalation path
- initial access

## Common Mistakes

- [ ]  Testing payloads before understanding context
- [ ]  Treating application discovery as exploitation
- [ ]  Missing authentication and access control issues
- [ ]  Ignoring API endpoints
- [ ]  Testing only visible form fields
- [ ]  Ignoring hidden parameters and JSON bodies
- [ ]  Ignoring file upload behaviour
- [ ]  Ignoring authenticated functionality
- [ ]  Not checking for product-specific exploit paths
- [ ]  Duplicating notes instead of linking to the primary note
