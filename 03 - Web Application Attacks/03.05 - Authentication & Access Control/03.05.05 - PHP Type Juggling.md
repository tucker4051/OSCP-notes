## Overview

PHP uses **loose comparison** when the `==` operator is used.  
Loose comparison attempts to convert both operands to a **common data type** before comparing them.

This behaviour can introduce serious security vulnerabilities when developers compare:

- authentication tokens
- password hashes
- API keys
- session values
- security flags

using `==` instead of strict comparison (`===`).

One of the most well-known issues arising from this behaviour is the **Magic Hash** problem.

---

# 1. Loose vs Strict Comparison

PHP provides two comparison operators:

### Loose comparison
```php
==
```

Performs **type juggling** before comparison.

Example:

```php
var_dump("5" == 5);
```

Result:

```text
true
```

PHP converts both values to the same type before comparing.

---

### Strict comparison
```php
===
```

Checks:

- value
- type

Example:

```php
var_dump("5" === 5);
```

Result:

```text
false
```

Because:

- `"5"` → string
- `5` → integer

---

# 2. What is Type Juggling

Type juggling is PHP automatically converting variables between types when required.

Examples:

| Value 1 | Value 2 | Result |
|--------|--------|-------|
| "10" | 10 | true |
| "0" | false | true |
| "" | false | true |
| "abc" | 0 | true |
| "0e12345" | 0 | true |

This behaviour becomes dangerous when user-controlled input is compared to sensitive values.

---

# 3. Magic Hash Behaviour

A **Magic Hash** is a string that looks like scientific notation for zero.

Format:

```text
0e + digits only
```

Example:

```text
0e123456789
```

PHP interprets this as:

```text
0 × 10^123456789
```

Which evaluates to:

```text
0
```

Therefore:

```php
"0e123456789" == "0e987654321"
```

Evaluates to:

```php
true
```

Because both are converted to numeric zero.

---

# 4. Why This Is Dangerous

Many applications store hashes such as:

- MD5
- SHA1
- SHA256

If a stored hash happens to begin with:

```text
0e...
```

and the developer compares hashes using:

```php
if($user_input == $stored_hash)
```

then **any other magic hash** may pass validation.

This allows authentication bypass without knowing the real hash value.

---

# 5. Example Vulnerable Code

```php
$stored_hash = md5("password123");

if($_POST['token'] == $stored_hash){
    echo "Access granted";
}
```

If `$stored_hash` equals something like:

```text
0e830400451993494058024219903391
```

Then the attacker only needs to supply another value that produces:

```text
0e...
```

PHP converts both to:

```text
0
```

Comparison succeeds.

### API authentication


# 9. Exploitation Workflow

## Step 1 – Identify comparison logic

Look for areas where hashes or tokens are compared using:

```php
==
```

instead of:

```php
===
```

Common locations:

- login endpoints
- API authentication
- password reset functionality
- session validation
- token validation

---

## Step 2 – Identify hash format

Look for values that resemble:

```text
32 hex chars → MD5
40 hex chars → SHA1
64 hex chars → SHA256
```

Example:

```text
5f4dcc3b5aa765d61d8327deb882cf99
```

---

## Step 3 – Test magic hashes

Submit values known to produce:

```text
0e + digits
```

Use:

- Burp Intruder
- ffuf
- custom wordlists

---

## Step 4 – Observe response behaviour

Successful bypass indicators:

- 200 OK
- login success
- token accepted
- access granted
- privilege escalation

# 12. Why This Happens Internally

When PHP compares:

```php
"0e123456" == "0e987654"
```

PHP converts both values to floats:

```text
0e123456 → 0
0e987654 → 0
```

Result:

```php
0 == 0
```

Which is:

```php
true
```

---

# 13. Prevention

Always use strict comparison for sensitive checks.

Correct implementation:

```php
if($user_input === $stored_hash)
```

---

## Additional protections

### Type enforcement

Ensure values are treated strictly as strings:

```php
hash_equals($stored_hash, $user_input)
```

---

### Input validation

Ensure tokens:

- follow expected length
- contain expected character set

---

### Avoid loose comparisons entirely

Avoid:

```php
==
!=
```

Prefer:

```php
===
!==
```

---

# 14. Secure Example

```php
if(hash_equals($stored_hash, $user_input)){
    echo "Access granted";
}
```

`hash_equals` prevents:

- timing attacks
- type juggling issues

---

# 15. Key Takeaways

- PHP loose comparison performs automatic type conversion
- strings beginning with `0e` followed only by digits evaluate to numeric zero
- attackers can exploit this behaviour to bypass authentication checks
- the issue commonly affects MD5 and SHA-based token comparisons
- magic hashes allow attackers to authenticate without knowing the real token
- always use strict comparison or secure hash comparison functions
