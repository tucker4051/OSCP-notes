## Overview

**Plink** (PuTTY Link) is a Windows command-line SSH client included with the PuTTY suite.

It provides functionality similar to Linux SSH, including:

- dynamic port forwarding
- local port forwarding
- remote port forwarding
- SOCKS proxy creation

Plink is especially useful when:

- working from a Windows attack host
- pivoting through a Windows foothold
- living off the land (LotL)
- PuTTY is already installed on the compromised system
- installing new tools may trigger alerts

Before Windows included a native OpenSSH client, PuTTY was the most common SSH utility available on Windows systems, making Plink a realistic pivoting tool in many environments.

---

# Why Use Plink?

In many penetration testing scenarios:

- you may compromise a Windows host
- the host may be moderately locked down
- downloading additional tooling may trigger detection
- PuTTY may already be installed
- Plink may already be present on disk or accessible on a network share

Using Plink allows you to:

- avoid introducing unfamiliar binaries
- operate using existing tooling
- maintain a lower operational footprint
- create SSH tunnels similar to Linux environments

Plink is also useful if your **primary attack platform is Windows** instead of Linux.

---

# What Plink Can Do

Plink supports the same pivoting functionality as OpenSSH:

| Capability | Description |
|-----------|------------|
| Dynamic port forwarding | create a SOCKS proxy |
| Local port forwarding | forward a remote service to local machine |
| Remote port forwarding | expose a local service remotely |
| SSH tunnelling | encrypt pivot traffic |
| authentication support | password or key-based login |

---

# Scenario

Assume:

- Windows attack host
- Ubuntu pivot host reachable via SSH
- internal network accessible through pivot
- need to route tools through pivot

We want to create a SOCKS proxy on Windows using Plink.

---

# Dynamic Port Forwarding with Plink

## Command

```cmd
plink -ssh -D 9050 ubuntu@10.129.15.50
```

## Breakdown

`-ssh`

Use SSH protocol.

`-D 9050`

Create a dynamic port forward (SOCKS proxy) on:

```text
127.0.0.1:9050
```

`ubuntu@10.129.15.50`

SSH login target:

- username: ubuntu
- pivot host IP: 10.129.15.50

---

# What This Command Does

This creates an SSH session between:

```text
Windows attack host --> Ubuntu pivot host
```

Once established:

- Plink listens locally on port `9050`
- this port acts as a SOCKS proxy
- tools configured to use this proxy can reach internal network resources through the pivot

---

# Traffic Flow

```text
tool on Windows --> SOCKS proxy (127.0.0.1:9050)
                   --> SSH tunnel
                      --> Ubuntu pivot host
                         --> internal network resources
```

The pivot host becomes a gateway into the internal network.

---

# Using Proxifier with Plink

Unlike Linux environments that commonly use `proxychains`, Windows environments often use **Proxifier**.

Proxifier allows Windows applications to use a SOCKS proxy without native proxy support.

This means even applications that do not normally support proxy configuration can be forced through the tunnel.

---

# Configure Proxifier

After starting Plink:

```cmd
plink -ssh -D 9050 ubuntu@10.129.15.50
```

configure Proxifier with:

```text
SOCKS Server: 127.0.0.1
Port: 9050
Protocol: SOCKS5 or SOCKS4
```

Once configured, Proxifier routes supported application traffic through the SOCKS proxy.

---

# Example Use Case: RDP Pivot

After configuring the SOCKS proxy, you can launch RDP via:

```cmd
mstsc.exe
```

Then connect to an internal system reachable only through the pivot.

Example:

```text
172.16.5.19
```

Proxifier ensures that the RDP traffic flows through:

```text
mstsc.exe --> Proxifier --> 127.0.0.1:9050 --> SSH tunnel --> pivot host --> internal target
```

# Living Off the Land Advantage

Plink is valuable in environments where:

- application whitelisting is enforced
- EDR flags unknown binaries
- file transfers are heavily monitored
- introducing tools raises suspicion

Because PuTTY is widely used:

- Plink is often trusted
- it may already exist on disk
- using it blends in with administrative behaviour

This aligns with a **LotL (Living off the Land)** approach.

# Alternative Plink Pivot Types

## Local Port Forwarding

Equivalent to:

```bash
ssh -L
```

Example:

```cmd
plink -ssh -L 3389:172.16.5.19:3389 ubuntu@10.129.15.50
```

This maps:

```text
localhost:3389 --> 172.16.5.19:3389
```

Useful for RDP pivoting without SOCKS.

---

## Remote Port Forwarding

Equivalent to:

```bash
ssh -R
```

Example:

```cmd
plink -ssh -R 8080:127.0.0.1:80 ubuntu@10.129.15.50
```

Useful when internal hosts need to connect back through a pivot.

---

# Typical Workflow

1. gain access to Windows host
2. discover PuTTY or plink.exe present
3. identify reachable pivot host via SSH
4. create dynamic SOCKS proxy using plink
5. configure Proxifier to use SOCKS proxy
6. run tools such as:

   - mstsc
   - browser
   - nmap (Windows)
   - custom tooling

7. access internal network resources via pivot

# Common Pitfalls

## Incorrect proxy configuration

Proxifier must point to:

```text
127.0.0.1:9050
```

## Firewall blocking SSH

Ensure port 22 is reachable from Windows host.

## Authentication issues

Credentials or SSH keys must be valid.

## SOCKS version mismatch

Ensure SOCKS version configured in Proxifier matches plink configuration.

## DNS leaks

Some applications may resolve DNS outside the proxy unless configured properly.

# Mental Model

Plink is essentially the Windows equivalent of:

```bash
ssh -D
```

It creates a secure tunnel from a Windows system into another network.

Applications on the Windows host can then send their traffic through that tunnel.

This allows the Windows system to behave like it is inside the internal network.

---

# Key Takeaways

- Plink is a Windows command-line SSH client included with PuTTY.
- It supports SOCKS proxies and port forwarding.
- It is useful for pivoting when working from Windows systems.
- Proxifier allows Windows applications to use the SOCKS tunnel.
- Plink is valuable for living-off-the-land pivoting where introducing new tools is risky.
- It provides functionality equivalent to Linux SSH tunnelling.
